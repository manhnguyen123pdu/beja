SERVICE LÀ GÌ?

1. Định nghĩa

Service là tầng chứa logic nghiệp vụ (business logic) của ứng dụng. Nó nằm giữa Controller và Repository, chịu trách nhiệm xử lý các quy tắc nghiệp vụ trước khi tương tác với database.

2. Vai trò của Service

Controller (API)
← Nhận request, trả response
Service (Xử lý logic)
← Gọi các phương thức JPA
Repository (Thao tác dữ liệu)
← Lưu trữ
Database (MySQL)

3. Tại sao cần Service?

Vấn đề Giải pháp từ Service
Controller quá nhiều code Controller chỉ gọi Service, không xử lý logic phức tạp
Logic nghiệp vụ phức tạp Service tập trung xử lý các quy tắc nghiệp vụ
Tái sử dụng code Nhiều Controller có thể dùng chung một Service
Kiểm tra dữ liệu Service kiểm tra tính hợp lệ trước khi lưu
Transaction Service quản lý transaction (thành công/có lỗi thì rollback)
Ví dụ: Khi đăng ký user mới, Service sẽ:
  • Kiểm tra email đã tồn tại chưa
  • Kiểm tra mật khẩu có đủ mạnh không
  • Mã hóa mật khẩu trước khi lưu
  • Gửi email xác nhận (nếu cần)
  • Lưu user vào database qua Repository

@SERVICE ANNOTATION

1. Định nghĩa

@Service là annotation đánh dấu một class là Service trong Spring Boot. Nó là một stereotype annotation (cùng với @Controller, @Repository, @Component).

2. Cú pháp

@Service Syntax
package com.example.jpa.service;

import org.springframework.stereotype.Service;

@Service
public class TenService {
    // Các phương thức xử lý logic
    public String sayHello() {
        return "Hello from Service";
    }
}

3. Ví dụ đơn giản

UserService.java - Simple Example
package com.example.jpa.service;

import org.springframework.stereotype.Service;

@Service
public class UserService {

    public String sayHello() {
        return "Hello from UserService";
    }
}

4. Giải thích

  • @Service giúp Spring Boot biết đây là Service
  • Spring sẽ tự động tạo bean (đối tượng) của class này
  • Có thể @Autowired Service vào Controller
  • Service thường chứa các phương thức xử lý logic nghiệp vụ

UserService.java - Phần 3: Thêm mới user
/**
* Thêm user mới
* @param user Thông tin user cần thêm
* @return User đã được lưu (có ID)
*/
@Transactional // Đảm bảo toàn vẹn dữ liệu
public User createUser(User user) {
    // Kiểm tra dữ liệu đầu vào
    validateUser(user);
    
    // Kiểm tra email đã tồn tại chưa
    if (userRepository.existsByEmail(user.getEmail())) {
        throw new RuntimeException("Email " + user.getEmail() + " đã được sử dụng!");
    }
    
    // Lưu user vào database
    return userRepository.save(user);
}

/**
* Kiểm tra dữ liệu user có hợp lệ không
*/
private void validateUser(User user) {
    if (user.getName() == null || user.getName().trim().isEmpty()) {
        throw new RuntimeException("Tên user không được để trống");
    }
    
    if (user.getEmail() == null || !user.getEmail().contains("@")) {
        throw new RuntimeException("Email không hợp lệ");
    }
    
    if (user.getPassword() == null || user.getPassword().length() < 6) {
        throw new RuntimeException("Mật khẩu phải có ít nhất 6 ký tự");
    }
    
    if (user.getAge() == null || user.getAge() < 0 || user.getAge() > 150) {
        throw new RuntimeException("Tuổi không hợp lệ");
    }
}
UserService.java - Phần 4: Cập nhật user
/**
* Cập nhật user
* @param id ID của user cần cập nhật
* @param userDetails Thông tin mới
* @return User đã được cập nhật
*/
@Transactional
public User updateUser(Long id, User userDetails) {
    // Tìm user cần cập nhật
    User existingUser = findUserById(id);
    
    // Kiểm tra email mới có bị trùng không (nếu thay đổi email)
    if (!existingUser.getEmail().equals(userDetails.getEmail())
            && userRepository.existsByEmail(userDetails.getEmail())) {
        throw new RuntimeException("Email " + userDetails.getEmail() + " đã được sử dụng!");
    }
    
    // Cập nhật thông tin
    existingUser.setName(userDetails.getName());
    existingUser.setEmail(userDetails.getEmail());
    existingUser.setPassword(userDetails.getPassword());
    existingUser.setAge(userDetails.getAge());
    existingUser.setPhoneNumber(userDetails.getPhoneNumber());
    
    // Lưu thay đổi (JPA tự động update vì đã có ID)
    return userRepository.save(existingUser);
}

@TRANSACTIONAL - QUẢN LÝ GIAO DỊCH

1. Transaction là gì?

Transaction (giao dịch) là một nhóm các thao tác phải được thực hiện hoặc tất cả thành công, hoặc tất cả thất bại.

2. Ví dụ về transaction

Ví dụ chuyển tiền
// Chuyển tiền: A chuyển 100k cho B
// Nếu chỉ trừ tiền A mà không cộng cho B → sai
// Phải đảm bảo cả 2 đều thành công

@Transactional
public void transferMoney(Long fromId, Long toId, double amount) {
    accountRepository.withdraw(fromId, amount); // Trừ tiền A
    accountRepository.deposit(toId, amount); // Cộng tiền B
    // Nếu lệnh thứ 2 lỗi, lệnh 1 tự động rollback
}

3. Các thuộc tính của @Transactional

Thuộc tính Ý nghĩa Ví dụ
readOnly = true Chỉ đọc, không sửa dữ liệu (tối ưu cho SELECT) @Transactional(readOnly = true)
rollbackFor Chỉ định lỗi nào thì rollback rollbackFor = Exception.class
noRollbackFor Lỗi nào không rollback noRollbackFor = BusinessException.class
timeout Thời gian tối đa (giây) timeout = 30

4. Ví dụ sử dụng @Transactional

Các cách dùng @Transactional
@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    // Chỉ đọc - tối ưu hiệu năng
    @Transactional(readOnly = true)
    public List<User> getAllUsers() {
        return userRepository.findAll();
    }

    // Ghi dữ liệu - cần transaction
    @Transactional
    public User createUser(User user) {
        return userRepository.save(user);
    }

    // Với rollback cụ thể
    @Transactional(rollbackFor = Exception.class, timeout = 10)
    public void complexOperation() {
        // Nếu mất hơn 10 giây → rollback
        // Nếu có Exception → rollback
    }
}
Lưu ý:
  • Nên dùng @Transactional(readOnly = true) cho các phương thức chỉ đọc để tối ưu hiệu năng
  • Các phương thức thêm/sửa/xóa nên có @Transactional
  • Transaction thường được đặt ở tầng Service, không phải Controller hay Repository

UserService.java - Với Custom Exceptions
package com.example.jpa.service;

import com.example.jpa.exception.DuplicateEmailException;
import com.example.jpa.exception.InvalidUserDataException;
import com.example.jpa.exception.UserNotFoundException;
import com.example.jpa.model.User;
import com.example.jpa.repository.UserRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;

@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public User getUserById(Long id) {
        return userRepository.findById(id)
            .orElseThrow(() -> new UserNotFoundException(id));
    }

    @Transactional
    public User createUser(User user) {
        validateUser(user);
        
        if (userRepository.existsByEmail(user.getEmail())) {
            throw new DuplicateEmailException(user.getEmail());
        }
        
        return userRepository.save(user);
    }

    @Transactional
    public User updateUser(Long id, User userDetails) {
        User existingUser = getUserById(id);
        
        if (!existingUser.getEmail().equals(userDetails.getEmail())
                && userRepository.existsByEmail(userDetails.getEmail())) {
            throw new DuplicateEmailException(userDetails.getEmail());
        }
        
        existingUser.setName(userDetails.getName());
        existingUser.setEmail(userDetails.getEmail());
        existingUser.setPassword(userDetails.getPassword());
        existingUser.setAge(userDetails.getAge());
        existingUser.setPhoneNumber(userDetails.getPhoneNumber());
        
        return userRepository.save(existingUser);
    }

    private void validateUser(User user) {
        if (user.getName() == null || user.getName().trim().isEmpty()) {
            throw new InvalidUserDataException("Tên user không được để trống");
        }
        // ... các kiểm tra khác
    }
}

UserController.java - Gọi Service
package com.example.jpa.controller;

import com.example.jpa.model.User;
import com.example.jpa.service.UserService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/users")
public class UserController {

    // Inject Service, không inject Repository
    @Autowired
    private UserService userService;

    // GET: Lấy tất cả users
    @GetMapping
    public List<User> getAllUsers() {
        return userService.getAllUsers();
    }

    // GET: Lấy user theo ID
    @GetMapping("/{id}")
    public ResponseEntity<User> getUserById(@PathVariable Long id) {
        try {
            User user = userService.findUserById(id);
            return ResponseEntity.ok(user);
        } catch (RuntimeException e) {
            return ResponseEntity.notFound().build();
        }
    }

    // POST: Thêm user mới

TỔNG KẾT VỀ SERVICE

1. Các annotation chính

Annotation Mục đích
@Service Đánh dấu class là Service
@Autowired Inject dependency (Repository)
@Transactional Quản lý transaction

2. Quy tắc khi viết Service

STT Quy tắc Giải thích
1 Không để Controller gọi trực tiếp Repository Controller chỉ gọi Service, mọi logic phải qua Service
2 Xử lý exception trong Service Ném exception phù hợp với từng trường hợp lỗi
3 Validate dữ liệu Kiểm tra đầu vào trước khi lưu vào database
4 Dùng @Transactional cho các method ghi Đảm bảo toàn vẹn dữ liệu khi có lỗi
5 Tách biệt logic Mỗi method chỉ làm một việc, dễ bảo trì

3. Luồng xử lý hoàn chỉnh

  1. Client gửi request → Controller
  2. Controller nhận request, gọi Service
  3. Service xử lý logic:
    • Validate dữ liệu đầu vào
    • Kiểm tra điều kiện nghiệp vụ
    • Gọi Repository để thao tác database
  4. Repository thao tác với database
  5. Service trả kết quả về Controller
  6. Controller trả response về client
