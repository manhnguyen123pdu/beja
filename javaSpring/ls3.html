ÔN TẬP CÁC PHƯƠNG THỨC HTTP

1. Bảng tổng hợp

Phương thức Chức năng Idempotent Spring Annotation
GET Lấy dữ liệu Có @GetMapping
POST Thêm mới Không @PostMapping
PUT Cập nhật toàn bộ Có @PutMapping
PATCH Cập nhật một phần Không @PatchMapping
DELETE Xóa Có @DeleteMapping

2. Idempotent là gì?

Idempotent (tính chất lũy đẳng) nghĩa là gọi 1 lần hay nhiều lần đều cho kết quả giống nhau.

Phương thức Idempotent? Giải thích
GET ✅ Lấy dữ liệu 1 lần hay 10 lần đều như nhau
POST ❌ POST 2 lần → tạo 2 bản ghi khác nhau
PUT ✅ PUT 2 lần với cùng dữ liệu → kết quả cuối cùng giống nhau
DELETE ✅ DELETE 2 lần → lần đầu xóa, lần sau không tìm thấy nhưng trạng thái cuối vẫn là đã xóa

@PUTMAPPING - CẬP NHẬT DỮ LIỆU

1. Định nghĩa

@PutMapping là annotation dùng để xử lý các request HTTP PUT.

PUT là phương thức dùng để cập nhật toàn bộ dữ liệu của một tài nguyên.

2. Mục đích sử dụng

  • Cập nhật thông tin user
  • Sửa sản phẩm
  • Thay đổi mật khẩu
  • Chỉnh sửa bài viết

3. So sánh POST và PUT

Mục đích Thêm mới Cập nhật
Idempotent Không Có

8. Ví dụ 3: Cập nhật có kiểm tra tồn tại

Ví dụ
@PutMapping("/edit/{id}")
public String editUser(
    @PathVariable String id,
    @RequestBody User updateUser) {

    // Tìm vị trí của user trong List
    int index = -1;
    for (int i = 0; i < users.size(); i++) {
        if (users.get(i).getId().equals(id)) {
            index = i;
            break;
        }
    }

    // Nếu tìm thấy thì cập nhật
    if (index != -1) {
        updateUser.setId(id); // Đảm bảo giữ nguyên ID
        users.set(index, updateUser);
        return "Cập nhật thành công user ID: " + id;
    } else {
        return "Lỗi: Không tìm thấy user ID: " + id;
    }
}

RESPONSEENTITY - TRẢ VỀ CHUYÊN NGHIỆP

1. Vấn đề với cách trả về cũ

Cách trả về String ở Bài 2:

Cách trả về cũ
@GetMapping("/getAll")
public List<User> getAllUsers() {
    return users; // Chỉ trả về dữ liệu
}

@DeleteMapping("/delete/{id}")
public String deleteUser(@PathVariable String id) {
    if (timThay) {
        return "Xóa thành công"; // Chỉ là String
    } else {
        return "Không tìm thấy"; // Cũng là String
    }

Nhược điểm:

  • Không trả về HTTP status code phù hợp
  • Client không biết request thành công hay thất bại
  • Thiếu thông tin về lỗi

2. HTTP Status Code là gì?

HTTP status code là mã trạng thái 3 chữ số server trả về cho client.

Các nhóm status code:

Mã Ý nghĩa Ví dụ
1xx Thông tin Hiếm dùng
2xx Thành công 200 OK, 201 Created
3xx Chuyển hướng 301 Moved Permanently
4xx Lỗi từ client 400 Bad Request, 404 Not Found
5xx Lỗi từ server 500 Internal Server Error

Các status code thường dùng trong REST API:

Mã Ý nghĩa Khi nào dùng
200 OK Thành công GET, PUT, DELETE thành công
201 Created Đã tạo POST tạo mới thành công
204 No Content Thành công nhưng không có dữ liệu DELETE thành công
400 Bad Request Request sai Thiếu thông tin, JSON sai
404 Not Found Không tìm thấy ID không tồn tại
500 Internal Server Error Lỗi hệ thống Exception trong code

3. ResponseEntity là gì?

ResponseEntity là một class trong Spring cho phép chúng ta kiểm soát toàn bộ HTTP response:

  • HTTP status code
  • Headers
  • Body (dữ liệu trả về)

4. Cú pháp ResponseEntity

Cách 1: Tạo trực tiếp

Cú pháp
return new ResponseEntity<>(dữLiệu, HttpStatus.MÃ_STATUS);

Cách 2: Dùng builder methods

Cú pháp
return ResponseEntity.status(HttpStatus.MÃ_STATUS).body(dữLiệu);
return ResponseEntity.ok(dữLiệu); // 200 OK
return ResponseEntity.status(404).body("Không tìm thấy");

5. Các phương thức thường dùng của ResponseEntity

Phương thức Ý nghĩa Ví dụ
ok() 200 OK ResponseEntity.ok(user)
status(201) Custom status ResponseEntity.status(201).body(user)
badRequest() 400 Bad Request ResponseEntity.badRequest().body("Lỗi")
notFound() 404 Not Found ResponseEntity.notFound().build()
noContent() 204 No Content ResponseEntity.noContent().build()

6. Ví dụ 1: GET với ResponseEntity

Ví dụ
@GetMapping("/getAll")
public ResponseEntity<List<User>> getAllUsers() {
    // Trả về 200 OK kèm danh sách users
    return ResponseEntity.ok(users);
}

7. Ví dụ 2: GET theo ID có kiểm tra

Ví dụ
@GetMapping("/getUser/{id}")
public ResponseEntity<?> getUserById(@PathVariable String id) {
    // Tìm user trong List
    for (User user : users) {
        if (user.getId().equals(id)) {
            // Tìm thấy → 200 OK
            return ResponseEntity.ok(user);
        }
    }

    // Không tìm thấy → 404 Not Found
    return ResponseEntity.status(404).body("Không tìm thấy user ID: " + id);
}

Giải thích:

  • ResponseEntity<?>: ? nghĩa là có thể trả về nhiều kiểu khác nhau (User hoặc String)
  • Nếu tìm thấy: trả về 200 với dữ liệu User
  • Nếu không tìm thấy: trả về 404 với thông báo lỗi

8. Ví dụ 3: POST với ResponseEntity

Ví dụ
@PostMapping("/addUser")
public ResponseEntity<String> addUser(@RequestBody User user) {
    // Kiểm tra dữ liệu đầu vào
    if (user.getName() == null || user.getName().trim().isEmpty()) {
        // 400 Bad Request nếu thiếu tên
        return ResponseEntity.badRequest().body("Tên user không được để trống");
    }

    if (user.getEmail() == null || !user.getEmail().contains("@")) {
        // 400 Bad Request nếu email không hợp lệ
        return ResponseEntity.badRequest().body("Email không hợp lệ");
    }

    // Thêm user
    users.add(user);

    // 201 Created - Thêm thành công
    return ResponseEntity.status(201).body("Đã thêm user: " + user.getName());
}

9. Ví dụ 4: PUT với ResponseEntity

Ví dụ
@PutMapping("/updateUser/{id}")
public ResponseEntity<?> updateUser(
    @PathVariable String id,
    @RequestBody User updateUser) {

    // Tìm user cần cập nhật
    for (int i = 0; i < users.size(); i++) {
        if (users.get(i).getId().equals(id)) {
            // Giữ nguyên ID, cập nhật các thông tin khác
            updateUser.setId(id);
            users.set(i, updateUser);

            // 200 OK - Cập nhật thành công
            return ResponseEntity.ok(updateUser);
        }
    }

    // 404 Not Found - Không tìm thấy user
    return ResponseEntity.status(404).body("Không tìm thấy user ID: " + id);
}

10. Ví dụ 5: DELETE với ResponseEntity

Ví dụ
@DeleteMapping("/deleteUser/{id}")
public ResponseEntity<String> deleteUser(@PathVariable String id) {
    // Thử xóa user
    boolean removed = users.removeIf(user -> user.getId().equals(id));

    if (removed) {
        // 200 OK - Xóa thành công
        return ResponseEntity.ok("Đã xóa user ID: " + id);
    } else {
        // 404 Not Found - Không tìm thấy
        return ResponseEntity.status(404).body("Không tìm thấy user ID: " + id);
    }
}

CORS - KẾT NỐI VỚI FRONTEND

1. CORS là gì?

CORS (Cross-Origin Resource Sharing) là cơ chế cho phép hoặc chặn các request từ domain khác gọi đến API của bạn.

2. Vấn đề: Same-Origin Policy

Trình duyệt có chính sách bảo mật gọi là Same-Origin Policy:

Chỉ cho phép request cùng origin (cùng protocol + domain + port)

Origin là gì?

Origin
http://localhost:8080
[protocol] [domain] [port]

Ví dụ:

URL Origin So sánh
http://localhost:8080/users http://localhost:8080 Gốc
http://localhost:5500/index.html http://localhost:5500 Khác port → khác origin
https://localhost:8080/users https://localhost:8080 Khác protocol → khác origin
http://127.0.0.1:8080/users http://127.0.0.1:8080 Khác domain → khác origin

3. Tình huống thực tế

Kết quả: Trình duyệt chặn request, hiện lỗi CORS

4. Lỗi CORS trông như thế nào?

Trong console của trình duyệt:

Lỗi CORS
Access to XMLHttpRequest at 'http://localhost:8080/user/getAll'
from origin 'http://localhost:3000' has been blocked by CORS policy

5. Giải pháp: @CrossOrigin

@CrossOrigin là annotation trong Spring Boot cho phép các domain khác gọi API.

Cú pháp:

Cú pháp
@CrossOrigin(origins = "địa-chỉ-được-phép")

Các cách dùng:

Cách 1: Cho phép tất cả (dùng khi học/test)

@CrossOrigin("*")
@CrossOrigin(origins = "*")
@RestController
@RequestMapping("/user")
public class UserController {
    // ...
}

Cách 2: Cho phép domain cụ thể (dùng khi deploy)

@CrossOrigin cụ thể
@CrossOrigin(origins = "http://localhost:3000")
@RestController
@RequestMapping("/user")
public class UserController {
    // ...
}

Cách 3: Cho phép nhiều domain

Nhiều domain
@CrossOrigin(origins = {"http://localhost:3000", "http://127.0.0.1:5500"})
@RestController
@RequestMapping("/user")
public class UserController {
    // ...
}

Cách 4: Cho phép từng method riêng

Method-level CORS
@RestController
@RequestMapping("/user")
public class UserController {

    @CrossOrigin(origins = "http://localhost:3000")
    @GetMapping("/getAll")
    public List<User> getAllUsers() {
        return users;
    }

    // Method này không cho phép CORS
    @PostMapping("/addUser")
    public String addUser(@RequestBody User user) {
        users.add(user);
        return "Đã thêm user";
    }
}

6. Cơ chế hoạt động của @CrossOrigin

Khi thêm @CrossOrigin(origins = "*"), Spring Boot tự động thêm header vào response:

HTTP Header
Access-Control-Allow-Origin: *

Trình duyệt thấy header này → Cho phép request.

7. Giá trị origins

Giá trị Ý nghĩa
"*" Cho phép tất cả domain
"http://localhost:3000" Chỉ cho phép domain này
{"http://a.com", "http://b.com"} Cho phép nhiều domain

2. UserController.java hoàn chỉnh

UserController.java
package com.example.demo.controller;

import com.example.demo.model.User;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@CrossOrigin(origins = "*") // Cho phép mọi domain gọi API (khi học)
@RestController
@RequestMapping("/user")
public class UserController {

    // List lưu trữ users
    private List<User> users = new ArrayList<>();

    // Khởi tạo dữ liệu mẫu
    public UserController() {
        users.add(new User("1", "Nguyen Van A", "a@gmail.com", "123456"));
        users.add(new User("2", "Tran Thi B", "b@gmail.com", "789012"));
        System.out.println("Đã khởi tạo " + users.size() + " user mẫu");
    }

    // ===== 1. GET - Lấy tất cả users =====
    @GetMapping("/getAll")
    public ResponseEntity<List<User>> getAllUsers() {
        return ResponseEntity.ok(users);
    }

    // ===== 2. GET - Lấy user theo ID =====
    @GetMapping("/getUser/{id}")
    public ResponseEntity<?> getUserById(@PathVariable String id) {
        for (User user : users) {
            if (user.getId().equals(id)) {
                return ResponseEntity.ok(user);
            }
        }
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
                .body("Không tìm thấy user ID: " + id);
    }

    // ===== 3. GET - Tìm kiếm user theo tên =====
    @GetMapping("/search")
    public ResponseEntity<?> searchUsers(@RequestParam String name) {
        List<User> result = new ArrayList<>();

        for (User user : users) {
            if (user.getName().toLowerCase().contains(name.toLowerCase())) {
                result.add(user);
            }
        }

        if (result.isEmpty()) {
            return ResponseEntity.status(HttpStatus.NOT_FOUND)
                    .body("Không tìm thấy user nào có tên chứa: " + name);
        }

        return ResponseEntity.ok(result);
    }

    // ===== 4. POST - Thêm user mới =====
    @PostMapping("/addUser")
    public ResponseEntity<?> addUser(@RequestBody User user) {
        // Kiểm tra dữ liệu đầu vào
        if (user.getName() == null || user.getName().trim().isEmpty()) {
            return ResponseEntity.badRequest().body("Tên user không được để trống");
        }

        if (user.getEmail() == null || !user.getEmail().contains("@")) {
            return ResponseEntity.badRequest().body("Email không hợp lệ");
        }

        // Tạo ID mới nếu chưa có
        if (user.getId() == null || user.getId().isEmpty()) {
            String newId = String.valueOf(users.size() + 1);
            user.setId(newId);
        }

        users.add(user);

        return ResponseEntity.status(HttpStatus.CREATED)
                .body("Đã thêm user: " + user.getName());
    }

    // ===== 5. PUT - Cập nhật user =====
    @PutMapping("/updateUser/{id}")
    public ResponseEntity<?> updateUser(
        @PathVariable String id,
        @RequestBody User updateUser) {

        // Tìm user cần cập nhật
        for (int i = 0; i < users.size(); i++) {
            if (users.get(i).getId().equals(id)) {
                // Giữ nguyên ID, cập nhật các thông tin khác
                updateUser.setId(id);
                users.set(i, updateUser);

                return ResponseEntity.ok(updateUser);
            }
        }

        return ResponseEntity.status(HttpStatus.NOT_FOUND)
                .body("Không tìm thấy user ID: " + id);
    }

    // ===== 6. DELETE - Xóa user =====
    @DeleteMapping("/deleteUser/{id}")
    public ResponseEntity<?> deleteUser(@PathVariable String id) {
        boolean removed = users.removeIf(user -> user.getId().equals(id));

        if (removed) {
            return ResponseEntity.ok("Đã xóa user ID: " + id);
        } else {
            return ResponseEntity.status(HttpStatus.NOT_FOUND)
                .body("Không tìm thấy user ID: " + id);
        }
    }

    // ===== 7. GET - Lấy tổng số user =====
    @GetMapping("/count")
    public ResponseEntity<?> getCount() {
        return ResponseEntity.ok("Tổng số user: " + users.size());
    }

    // ===== 8. DELETE - Xóa tất cả users =====
    @DeleteMapping("/deleteAll")
    public ResponseEntity<?> deleteAllUsers() {
        users.clear();
        return ResponseEntity.ok("Đã xóa tất cả users");
    }
}
