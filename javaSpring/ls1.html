SPRING FRAMEWORK VÀ SPRING BOOT

2.1. Spring Framework là gì?

Spring Framework là một framework mạnh mẽ của Java, giúp lập trình viên xây dựng ứng dụng dễ dàng hơn.

Các tính năng chính của Spring Framework:

Tính năng Mô tả Ví dụ
Dependency Injection Tự động tạo và quản lý các đối tượng Không cần new đối tượng
AOP Lập trình hướng khía cạnh Logging, bảo mật
Transaction Management Quản lý giao dịch Rollback khi lỗi
Web MVC Xây dựng ứng dụng web Controller, View

Ví dụ trước và sau khi dùng Spring:

❌ Không dùng Spring:

Java - Không Spring
public class UserService {
    private UserRepository userRepository;
    
    public UserService() {
        // Phải tự tạo đối tượng
        userRepository = new UserRepository();
    }
}

✅ Dùng Spring:

Java - Với Spring
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository; // Spring tự tạo và inject
}

2.2. Spring Boot là gì?

Spring Boot là một dự án con của Spring Framework, được phát triển để đơn giản hóa việc tạo ứng dụng Spring.

Vấn đề trước khi có Spring Boot:

  • Phải cấu hình rất nhiều file XML
  • Phải tự cài đặt và cấu hình server
  • Phải tự quản lý phiên bản thư viện
  • Mất nhiều thời gian cho cấu hình

Spring Boot giải quyết:

  • ✅ Tự động cấu hình - Không cần file XML
  • ✅ Server nhúng - Có sẵn Tomcat, không cần cài đặt
  • ✅ Starter dependencies - Quản lý phiên bản tự động
  • ✅ Cấu hình đơn giản - Chỉ cần file application.properties

2.3. So sánh Spring và Spring Boot

Tiêu chí Spring Framework Spring Boot
Cấu hình Phức tạp, nhiều XML Đơn giản, tự động
Server Phải cài Tomcat riêng Có Tomcat nhúng sẵn
Khởi tạo Mất nhiều thời gian Spring Initializr
Dependencies Tự quản lý phiên bản Starter quản lý sẵn
Thời gian học Lâu Nhanh hơn

2.4. Lợi ích của Spring Boot

  • Giảm thiểu cấu hình phức tạp - Không cần file XML, tự động cấu hình dựa trên dependencies
  • Tích hợp server sẵn - Có Tomcat, Jetty nhúng, chạy ứng dụng bằng 1 lệnh
  • Phát triển REST API nhanh chóng - Chỉ cần vài annotation, tự động chuyển Object <-> JSON
  • Cộng đồng lớn - Nhiều tài liệu, video hướng dẫn, dễ tìm giải pháp khi gặp lỗi
  • Sẵn sàng cho production - Có sẵn tính năng monitoring, dễ dàng cấu hình cho môi trường thật

2. Giải thích chi tiết từng thành phần

2.1. File pom.xml - Quản lý dependencies

Vị trí: Thư mục gốc của dự án

Vai trò:

  • Khai báo các thư viện (dependencies) project sử dụng
  • Quản lý phiên bản của các thư viện
  • Cấu hình build (tạo file JAR/WAR)

Nội dung chi tiết:

pom.xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <modelVersion>4.0.0</modelVersion>
    
    <!-- Thông tin cơ bản của project -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.1.0</version>
    </parent>
    
    <!-- Metadata của project -->
    <groupId>com.example</groupId> <!-- Tên nhóm -->
    <artifactId>demo</artifactId> <!-- Tên project -->
    <version>0.0.1-SNAPSHOT</version> <!-- Phiên bản -->
    <name>demo</name> <!-- Tên hiển thị -->
    <description>Demo project for Spring Boot</description>
    
    <!-- Cấu hình Java -->
    <properties>
        <java.version>17</java.version> <!-- Dùng Java 17 -->
    </properties>
    
    <!-- Danh sách dependencies (thư viện) -->
    <dependencies>
        <!-- Spring Web: để tạo REST API -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
        
        <!-- Spring Boot Test: để viết test -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>
    
    <!-- Plugin để build -->
    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

2.2. File DemoApplication.java - File chạy chính

Vị trí: src/main/java/com/example/demo/DemoApplication.java

Vai trò:

  • Là entry point (điểm vào) của ứng dụng
  • Chứa hàm main() để chạy Spring Boot
  • Đánh dấu package gốc để Spring quét các component

Nội dung:

DemoApplication.java
package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication // Đánh dấu đây là ứng dụng Spring Boot
public class DemoApplication {

    public static void main(String[] args) {
        // Dòng lệnh này khởi động ứng dụng Spring Boot
        SpringApplication.run(DemoApplication.class, args);
    }

}

3.3. Viết code cho UserController

UserController.java
package com.example.demo.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController // Đánh dấu đây là REST Controller
@RequestMapping("/user") // Đường dẫn gốc: http://localhost:8080/user
public class UserController {

    // API 1: GET /user/getAllUser
    @GetMapping("/getAllUser")
    public String getAllUser() {
        return "All User";
    }

    // API 2: GET /user/addUser
    @GetMapping("/addUser")
    public String addUser() {
        return "Add User";
    }
}

3.4. Giải thích chi tiết từng dòng code

Dòng code Giải thích
@RestController Đánh dấu class này là REST Controller, mọi method trả về dữ liệu trực tiếp (không phải view)
@RequestMapping("/user") Định nghĩa đường dẫn gốc cho tất cả API trong class này
@GetMapping("/getAllUser") Xử lý request GET đến đường dẫn /user/getAllUser
public String getAllUser() Method trả về kiểu String
return "All User"; Nội dung trả về cho client

GIẢI THÍCH CHI TIẾT CÁC ANNOTATION

1. @SpringBootApplication

Vị trí: Trên class DemoApplication

Thành phần:

Công thức
@SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan

1.1. @Configuration

Đánh dấu class có thể chứa các bean (đối tượng do Spring quản lý). Tương đương với file XML cấu hình ngày xưa.

1.2. @EnableAutoConfiguration

Spring Boot tự động cấu hình dựa trên các dependencies. Ví dụ: có spring-boot-starter-web → tự động cấu hình Tomcat, DispatcherServlet.

1.3. @ComponentScan

Tự động quét các component trong package com.example.demo và các package con. Tìm các annotation: @Controller, @Service, @Repository, @Component.

2. @RestController là gì?

@RestController là annotation dùng trong Spring Boot để tạo REST API.

Nó thực chất là:

Công thức
@RestController = @Controller + @ResponseBody

2.1. @Controller là gì?

Vai trò: Đánh dấu class là Controller trong mô hình MVC.

MVC gồm:

  • Model: xử lý dữ liệu
  • View: giao diện (HTML)
  • Controller: nhận request và trả về view

Thường dùng khi bạn muốn trả về trang HTML.

Ví dụ:

@Controller
@Controller
public class UserController {

    @GetMapping("/home")
    public String home() {
        return "home"; // trả về file home.html
    }
}

Khi truy cập /home, Spring sẽ tìm file home.html và hiển thị giao diện. Mặc định @Controller trả về View.

2.2. @ResponseBody là gì?

Vai trò: Bắt Spring không trả về View, mà trả về dữ liệu trực tiếp.

Ví dụ:

@ResponseBody
@GetMapping("/hello")
@ResponseBody
public String hello() {
    return "Xin chào";
}

Khi gọi /hello, trình duyệt sẽ hiển thị: Xin chào. Không có file HTML nào được render.

Nếu trả về object:

@ResponseBody với Object
@GetMapping("/user")
@ResponseBody
public User getUser() {
    return new User(1, "Manh");
}

Spring sẽ tự động chuyển object thành JSON:

JSON Output
{
  "id": 1,
  "name": "Manh"
}

2.3. @RestController làm gì?

Thay vì viết:

@Controller + @ResponseBody
@Controller
@ResponseBody

Ta chỉ cần viết:

@RestController
@RestController

Ví dụ:

@RestController
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping
    public List<User> getAllUsers() {
        return userService.findAll();
    }
}

Tất cả method trong class này:

  • Đều trả về JSON
  • Không trả về HTML
  • Không cần viết @ResponseBody cho từng method

So sánh nhanh:

Annotation Đặc điểm
@Controller Trả về HTML. Nếu muốn trả về JSON phải thêm @ResponseBody
@ResponseBody Trả về dữ liệu (JSON, text...), không trả về View
@RestController Chuyên dùng để làm REST API, tự động trả về JSON

3. @RequestMapping

3.1. Mục đích

Dùng để ánh xạ (map) URL với class hoặc method trong Controller.

  • Có thể dùng cho tất cả các HTTP method: GET, POST, PUT, DELETE…
  • Có thể đặt ở:
    • Trên class → định nghĩa URL gốc
    • Trên method → định nghĩa URL cụ thể

3.2. Cú pháp

Dùng cơ bản:

Cú pháp
@RequestMapping("/duong-dan")

Dùng đầy đủ:

Cú pháp
@RequestMapping(
    value = "/duong-dan",
    method = RequestMethod.GET
)

3.3. Giải thích cú pháp

  • value: đường dẫn URL.
  • method: loại HTTP method (GET, POST, PUT, DELETE…).
  • RequestMethod.GET: chỉ định request phải là GET.
  • Nếu không ghi method, mặc định chấp nhận tất cả HTTP method.

3.4. Ví dụ ngắn

Ví dụ đặt ở class:

Ví dụ
@RestController
@RequestMapping("/users")
public class UserController {
}

Giải thích: Tất cả API trong class này sẽ bắt đầu bằng /users.

Ví dụ đặt ở method:

Ví dụ
@GetMapping
public String getAll() {
    return "Danh sách user";
}

Nếu viết bằng @RequestMapping:

Ví dụ
@RequestMapping(value = "/users", method = RequestMethod.GET)
public String getAll() {
    return "Danh sách user";
}

4. @GetMapping

4.1. Mục đích

Chuyên dùng để xử lý HTTP GET.

Là phiên bản rút gọn của: @RequestMapping(method = RequestMethod.GET)

Là cách viết hiện đại, gọn hơn, dễ đọc hơn.

4.2. Cú pháp

Cú pháp
@GetMapping("/duong-dan")

Hoặc:

Cú pháp
@GetMapping

4.3. Giải thích cú pháp

  • "/duong-dan": URL của API.
  • Chỉ xử lý request GET.
  • Không cần ghi method = RequestMethod.GET vì đã mặc định.

4.4. Ví dụ ngắn

Ví dụ
@RestController
@RequestMapping("/users")
public class UserController {

    @GetMapping
    public String getAllUsers() {
        return "Danh sách user";
    }

    @GetMapping("/1")
    public String getUserById() {
        return "User có id = 1";
    }
}

Kết quả:

  • GET /users → gọi getAllUsers()
  • GET /users/1 → gọi getUserById()

So sánh nhanh @RequestMapping và @GetMapping

Annotation Đặc điểm
@RequestMapping Dùng cho mọi HTTP method. Cần chỉ rõ method = RequestMethod.GET nếu muốn giới hạn. Linh hoạt hơn.
@GetMapping Chỉ dùng cho GET. Viết ngắn gọn. Dễ đọc, khuyến khích dùng trong Spring Boot hiện đại.

BÀI TẬP THỰC HÀNH

Bài tập 1: Tạo API chào mừng

Yêu cầu: Tạo API GET / trả về "Welcome to Spring Boot!"

Hướng dẫn:

  1. Trong UserController.java, thêm method mới
  2. Dùng @GetMapping("/")
  3. Return chuỗi "Welcome to Spring Boot!"

Đáp án:

UserController.java
@GetMapping("/")
public String welcome() {
    return "Welcome to Spring Boot!";
}

Kiểm tra: http://localhost:8080/

Bài tập 2: API có tham số

Yêu cầu: Tạo API /user/greet nhận tham số name từ URL, trả về "Hello, {name}!"

Ví dụ: /user/greet?name=Nguyen → "Hello, Nguyen!"

Hướng dẫn:

Dùng @RequestParam để lấy tham số từ URL

Đáp án:

UserController.java
@GetMapping("/greet")
public String greet(@RequestParam String name) {
    return "Hello, " + name + "!";
}

Kiểm tra: http://localhost:8080/user/greet?name=Nguyen

Bài tập 3: Đổi port server

Yêu cầu: Thay đổi port server sang 9090

Hướng dẫn:

  1. Mở file application.properties
  2. Thêm dòng: server.port=9090
  3. Chạy lại ứng dụng

Kiểm tra: http://localhost:9090/user/getAllUser
