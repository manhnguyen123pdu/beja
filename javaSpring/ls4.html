TỔNG QUAN VỀ DATABASE

1. Database là gì?

Database (cơ sở dữ liệu) là nơi lưu trữ dữ liệu có tổ chức, cho phép:

  • Lưu trữ lâu dài - không mất khi tắt ứng dụng
  • Truy xuất nhanh chóng - tối ưu cho tìm kiếm
  • Truy vấn phức tạp - tìm kiếm, lọc, sắp xếp
  • Đảm bảo tính toàn vẹn dữ liệu - ràng buộc, khóa ngoại

2. So sánh List và Database

Tiêu chí List (trong RAM) Database (MySQL)
Lưu trữ Tạm thời Vĩnh viễn
Mất khi restart Có Không
Tốc độ Rất nhanh Chậm hơn
Truy vấn phức tạp Khó (phải tự code) Dễ (SQL)
Dung lượng Ít (phụ thuộc RAM) Nhiều (phụ thuộc ổ cứng)
Bảo mật Không Có (phân quyền)

3. Các loại database phổ biến

3.1. SQL Database (Quan hệ)

  • MySQL: Phổ biến, miễn phí
  • PostgreSQL: Mạnh mẽ, nhiều tính năng
  • Oracle: Thương mại, dùng trong doanh nghiệp lớn
  • SQL Server: Của Microsoft

3.2. NoSQL Database (Phi quan hệ)

  • MongoDB: Lưu dạng JSON
  • Redis: Lưu trong bộ nhớ, tốc độ cao
  • Cassandra: Dữ liệu phân tán

Trong khóa học này: Chúng ta sẽ dùng MySQL.

JPA VÀ HIBERNATE

1. Vấn đề: Làm việc với database trong Java

Cách cũ: JDBC (Java Database Connectivity)

JDBC - Phải viết SQL thủ công
// Phải viết SQL thủ công
String sql = "INSERT INTO users(name, email) VALUES(?, ?)";
PreparedStatement stmt = connection.prepareStatement(sql);
stmt.setString(1, "Nguyen Van A");
stmt.setString(2, "a@gmail.com");
stmt.executeUpdate();

// Đọc dữ liệu
String query = "SELECT * FROM users WHERE id = ?";
ResultSet rs = stmt.executeQuery(query);
String name = rs.getString("name");

Nhược điểm:

  • Phải viết nhiều code lặp lại
  • Dễ sai chính tả tên cột
  • Khó bảo trì khi thay đổi cấu trúc bảng
  • Không an toàn (SQL injection)

2. JPA là gì?

JPA (Java Persistence API) là một đặc tả (specification) trong Java, định nghĩa cách ánh xạ các đối tượng Java sang các bảng trong database.

Nói đơn giản: JPA giúp chúng ta làm việc với database bằng Java object thay vì viết câu lệnh SQL.

2.1. Đặc tả là gì?

Đặc tả giống như một bản thiết kế, chỉ định nghĩa "làm gì" chứ không định nghĩa "làm thế nào".

Ví dụ:

  • JPA nói: "Phải có cách lưu object xuống database"
  • Nhưng JPA không cung cấp code cụ thể

2.2. Các khái niệm chính trong JPA

Khái niệm Mô tả
Entity Class Java ánh xạ với bảng database
EntityManager Quản lý vòng đời của Entity
JPQL Câu truy vấn trên Entity (giống SQL nhưng thao tác với object)
Persistence Unit Cấu hình kết nối database

3. Hibernate là gì?

Hibernate là một framework (thư viện cụ thể) triển khai (implementation) của JPA.

Nói đơn giản: Hibernate là hiện thực hóa bản thiết kế JPA thành code thật.

3.1. Mối quan hệ JPA và Hibernate

Mối quan hệ
JPA (Đặc tả - Chỉ là lý thuyết)
    ↑
    implements (triển khai)
    ↑
Hibernate (Framework cụ thể - Code thật)
    ↑
    Spring Boot tự động tích hợp
    ↑
Spring Data JPA (Spring Boot wrapper)

3.2. Ví dụ minh họa

JPA (đặc tả):

JPA Interface
// Đây chỉ là interface, không có code cụ thể
public interface EntityManager {
    void persist(Object entity); // Lưu object
    <T> T find(Class<T> entityClass, Object primaryKey); // Tìm theo ID
}

Hibernate (triển khai):

Hibernate Implementation
// Hibernate viết code cụ thể cho các phương thức JPA
public class HibernateEntityManager implements EntityManager {
    @Override
    public void persist(Object entity) {
        // Code cụ thể để lưu object xuống database
        generateSQLInsert();
        executeStatement();
        // ...
    }
}

4. Spring Data JPA là gì?

Spring Data JPA là module của Spring Boot, cung cấp các lớp và annotation giúp làm việc với JPA dễ dàng hơn.

4.1. Các thành phần chính của Spring Data JPA

Thành phần Vai trò Ví dụ
Entity Class Java ánh xạ với bảng database @Entity class User
Repository Interface cung cấp sẵn các phương thức CRUD UserRepository extends JpaRepository
Derived Query Methods Tự động tạo query dựa trên tên phương thức findByEmail(String email)
@Query Viết query tùy chỉnh @Query("SELECT u FROM User u WHERE u.age > ?1")

5. Tóm tắt

Luồng hoạt động
Spring Boot App
    ↓
Spring Data JPA (Wrapper dễ dùng)
    ↓
JPA (Đặc tả - Quy định chung)
    ↓
Hibernate (Triển khai cụ thể)
    ↓
JDBC (Kết nối database)
    ↓
MySQL (Database)

Lợi ích khi dùng Spring Data JPA:

  • Không cần viết SQL - Chỉ cần gọi phương thức
  • Tự động tạo bảng - Dựa trên Entity
  • Chuyển đổi tự động - Object <-> Database
  • Tối ưu câu lệnh - Hibernate tự tối ưu SQL
  • Hỗ trợ transaction - Đảm bảo toàn vẹn dữ liệu

ENTITY - ÁNH XẠ BẢNG TRONG DATABASE

1. Entity là gì?

Entity là class Java được ánh xạ với một bảng trong database. Mỗi instance (đối tượng) của Entity tương ứng với một dòng (row) trong bảng.

1.1. Ví dụ trực quan

Bảng USERS trong database:
+----+---------------+------------------+----------+
| id | full_name | email | password |
+----+---------------+------------------+----------+
| 1 | Nguyen Van A | a@gmail.com | 123456 |
| 2 | Tran Thi B | b@gmail.com | 789012 |
+----+---------------+------------------+----------+
Class User trong Java:
User{id=1, name="Nguyen Van A", email="a@gmail.com", password="123456"}
User{id=2, name="Tran Thi B", email="b@gmail.com", password="789012"}

1.2. Ánh xạ (Mapping)

Database (Bảng users) Java (Class User) Giải thích
Tên bảng: users Tên class: User @Table(name = "users")
Cột: id Thuộc tính: id @Column(name = "id")
Cột: full_name Thuộc tính: name @Column(name = "full_name")
Mỗi dòng dữ liệu Một đối tượng User Instance của class

CÁC ANNOTATION JPA CƠ BẢN

1. Bảng tổng hợp annotation

Annotation Mục đích Vị trí dùng
@Entity Đánh dấu class là Entity Trên class
@Table Chỉ định tên bảng trong database Trên class
@Id Đánh dấu khóa chính Trên thuộc tính
@GeneratedValue Tự động sinh giá trị cho khóa chính Trên thuộc tính
@Column Cấu hình cột trong bảng Trên thuộc tính
@Transient Không ánh xạ xuống database Trên thuộc tính
@Temporal Định dạng ngày tháng Trên thuộc tính Date
@Enumerated Ánh xạ Enum Trên thuộc tính Enum

2. @Entity - Đánh dấu class là Entity

Định nghĩa: @Entity là annotation đánh dấu một class Java sẽ được ánh xạ với một bảng trong database.

Cú pháp:

@Entity
import jakarta.persistence.Entity;

@Entity
public class TenClass {
    // Các thuộc tính
}

Giải thích:

  • Khi có @Entity, JPA sẽ quản lý class này
  • Mặc định, tên bảng trong database sẽ là tên class (viết thường)
  • Ví dụ: class User → bảng user

3. @Table - Chỉ định tên bảng

Định nghĩa: @Table dùng để chỉ định tên bảng trong database, nếu muốn khác tên class.

Cú pháp:

@Table
@Entity
@Table(name = "ten_bang_trong_database")
public class TenClass {
    // ...
}

Ví dụ:

Ví dụ @Table
import jakarta.persistence.Entity;
import jakarta.persistence.Table;

@Entity
@Table(name = "users") // Bảng trong database tên là "users"
public class User { // Class tên là "User"
    // ...
}

So sánh:

Không dùng @Table Có dùng @Table
@Entity class User → bảng user @Table(name = "users") → bảng users
Không thể đổi tên bảng Có thể đặt tên bảng tùy ý

4. @Id - Khóa chính

Định nghĩa: @Id đánh dấu thuộc tính là khóa chính (primary key) của bảng.

Cú pháp:

@Id
@Id
private KieuDuLieu tenThuocTinh;

Ví dụ:

Ví dụ @Id
import jakarta.persistence.Id;

@Entity
@Table(name = "users")
public class User {

    @Id
    private Long id; // Đây là khóa chính

    // Các thuộc tính khác
}

Giải thích:

  • Mỗi Entity bắt buộc phải có một thuộc tính đánh dấu @Id
  • Giá trị của khóa chính là duy nhất trong bảng
  • Thường dùng kiểu Long hoặc Integer cho khóa chính tự tăng

5. @GeneratedValue - Tự động sinh giá trị

Định nghĩa: @GeneratedValue dùng để tự động sinh giá trị cho khóa chính.

Cú pháp:

@GeneratedValue
@Id
@GeneratedValue(strategy = GenerationType.CHIEN_LUOC)
private Long id;

Các chiến lược (Strategy):

Chiến lược Mô tả Database hỗ trợ
GenerationType.IDENTITY Tự động tăng (AUTO_INCREMENT) MySQL, SQL Server, PostgreSQL
GenerationType.SEQUENCE Dùng sequence Oracle, PostgreSQL
GenerationType.TABLE Dùng bảng riêng để lưu giá trị Tất cả (nhưng chậm)
GenerationType.AUTO Để Hibernate tự chọn Mặc định

Ví dụ với MySQL (IDENTITY):

@GeneratedValue với IDENTITY
import jakarta.persistence.*;

@Entity
@Table(name = "users")
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY) // MySQL tự tăng
    private Long id;

    // Các thuộc tính khác
}

Giải thích:

  • Với strategy = GenerationType.IDENTITY:
  • MySQL sẽ tự động tạo giá trị id (1, 2, 3, ...)
  • Không cần gán giá trị cho id khi thêm mới
  • Tương ứng với AUTO_INCREMENT trong MySQL

6. @Column - Cấu hình cột

Định nghĩa: @Column dùng để cấu hình chi tiết cho một cột trong database.

Cú pháp:

@Column
@Column(
    name = "ten_cot",
    nullable = true/false,
    unique = true/false,
    length = 255,
    columnDefinition = "định_nghĩa_sql"
)
private KieuDuLieu tenThuocTinh;

Các thuộc tính của @Column:

Thuộc tính Ý nghĩa Ví dụ
name Tên cột trong database name = "full_name"
nullable Có được phép null không? nullable = false (không null)
unique Giá trị có duy nhất không? unique = true (không trùng)
length Độ dài tối đa (với String) length = 100
columnDefinition Định nghĩa SQL trực tiếp columnDefinition = "TEXT"
insertable Có tham gia INSERT không? Mặc định true
updatable Có tham gia UPDATE không? Mặc định true

Ví dụ 1: Cấu hình cơ bản

@Column cơ bản
@Column(name = "full_name", nullable = false, length = 100)
private String name;
// SQL tương ứng: full_name VARCHAR(100) NOT NULL

Ví dụ 2: Cột unique

@Column unique
@Column(unique = true, nullable = false)
private String email;
// SQL tương ứng: email VARCHAR(255) NOT NULL UNIQUE

Ví dụ 3: Cột có thể null

@Column nullable
@Column(name = "phone_number", length = 15)
private String phoneNumber; // nullable = true (mặc định)
// SQL tương ứng: phone_number VARCHAR(15)

Ví dụ 4: Cột với định nghĩa đặc biệt

@Column với columnDefinition
@Column(columnDefinition = "TEXT")
private String description;

@Column(columnDefinition = "DECIMAL(10,2)")
private Double price;
// SQL tương ứng: description TEXT, price DECIMAL(10,2)

So sánh khi có và không có @Column:

Không dùng @Column Có dùng @Column
Tên cột = tên thuộc tính Đặt tên cột tùy ý
nullable = true (mặc định) Có thể set nullable = false
length = 255 (mặc định) Có thể set độ dài khác
Không thể set unique Có thể set unique = true

VÍ DỤ ENTITY

1. Entity User hoàn chỉnh (không dùng Lombok)

User.java - Không Lombok
package com.example.jpa.model;

import jakarta.persistence.*;
import java.time.LocalDateTime;

@Entity // 1. Đánh dấu đây là Entity
@Table(name = "users") // 2. Tên bảng là "users"
public class User {

    // ===== 3. KHÓA CHÍNH =====
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY) // MySQL tự tăng
    private Long id;

    // ===== 4. CÁC THUỘC TÍNH =====

    @Column(name = "full_name", nullable = false, length = 100)
    private String name; // Họ tên - bắt buộc, tối đa 100 ký tự

    @Column(unique = true, nullable = false)
    private String email; // Email - duy nhất, bắt buộc

    @Column(nullable = false)
    private String password; // Mật khẩu - bắt buộc

    @Column(name = "phone_number", length = 15)
    private String phoneNumber; // Số điện thoại - có thể null

    @Column(nullable = false)
    private Integer age; // Tuổi - bắt buộc

    @Column(name = "created_at")
    private LocalDateTime createdAt; // Thời gian tạo

    @Column(name = "updated_at")
    private LocalDateTime updatedAt; // Thời gian cập nhật

    // ===== 5. CONSTRUCTOR =====

    // Constructor không tham số (bắt buộc cho JPA)
    public User() {
    }

    // Constructor có tham số (tiện lợi)
    public User(String name, String email, String password, Integer age) {
        this.name = name;
        this.email = email;
        this.password = password;
        this.age = age;
        this.createdAt = LocalDateTime.now();
        this.updatedAt = LocalDateTime.now();
    }

    // ===== 6. GETTER VÀ SETTER =====

    public Long getId() {
        return id;
    }

    // Không có setId cho id vì nó tự động sinh

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }

    public String getPhoneNumber() {
        return phoneNumber;
    }

    public void setPhoneNumber(String phoneNumber) {
        this.phoneNumber = phoneNumber;
    }

    public Integer getAge() {
        return age;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    public LocalDateTime getCreatedAt() {
        return createdAt;
    }

    public void setCreatedAt(LocalDateTime createdAt) {
        this.createdAt = createdAt;
    }

    public LocalDateTime getUpdatedAt() {
        return updatedAt;
    }

    public void setUpdatedAt(LocalDateTime updatedAt) {
        this.updatedAt = updatedAt;
    }

    // ===== 7. CÁC PHƯƠNG THỨC TỰ ĐỘNG =====

    // Tự động gọi trước khi lưu lần đầu
    @PrePersist
    protected void onCreate() {
        createdAt = LocalDateTime.now();
        updatedAt = LocalDateTime.now();
    }

    // Tự động gọi trước khi cập nhật
    @PreUpdate
    protected void onUpdate() {
        updatedAt = LocalDateTime.now();
    }

    // ===== 8. toString() =====
    @Override
    public String toString() {
        return "User{" +
                "id=" + id +
                ", name='" + name + '\'' +
                ", email='" + email + '\'' +
                ", age=" + age +
                '}'; // Không in password vì bảo mật
    }
}

3. Các annotation bổ sung khác

@Transient - Không lưu xuống database

@Transient
@Transient
private String confirmPassword; // Trường này không được lưu vào database

Dùng khi: Cần trường để xử lý logic nhưng không cần lưu (xác nhận mật khẩu, tính toán tạm thời).

@Enumerated - Ánh xạ Enum

@Enumerated
public enum UserRole {
    ADMIN, USER, MANAGER
}

@Entity
public class User {
    // ...

    @Enumerated(EnumType.STRING) // Lưu tên enum: "ADMIN", "USER"
    private UserRole role;

    @Enumerated(EnumType.ORDINAL) // Lưu thứ tự: 0, 1, 2 (không nên dùng)
    private UserRole role2;
}

@Lob - Lưu dữ liệu lớn

@Lob
@Lob
private String longText; // Lưu văn bản dài (TEXT trong MySQL)

@Lob
private byte[] image; // Lưu file nhị phân (BLOB)

4. Quy tắc khi tạo Entity

Quy tắc bắt buộc:

STT Quy tắc Lý do
1 Có @Entity Đánh dấu là Entity
2 Có ít nhất một @Id Mỗi bảng phải có khóa chính
3 Có constructor không tham số JPA cần để tạo object
4 Có getter cho mọi thuộc tính Để đọc dữ liệu
5 Không để final class Hibernate cần proxy

Quy tắc nên làm:

STT Quy tắc Lý do
1 Dùng Long cho ID Hỗ trợ số lớn, null khi chưa lưu
2 Đặt tên bảng số nhiều users, products, orders
3 Dùng nullable = false cho trường bắt buộc Database tự kiểm tra
4 Dùng unique = true cho trường duy nhất Đảm bảo không trùng
5 Có createdAt, updatedAt Theo dõi thời gian
