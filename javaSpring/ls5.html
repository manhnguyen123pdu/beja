REPOSITORY LÀ GÌ?

1. Định nghĩa

Repository là interface cung cấp các phương thức thao tác với database (CRUD) mà không cần viết SQL. Nó là cầu nối giữa Entity và database.

2. Vai trò của Repository

Controller (API)
↓
Service (Xử lý logic)
↓
Repository (Thao tác dữ liệu) ← BẠN ĐANG Ở ĐÂY
↓
Entity (Ánh xạ bảng)
↓
Database (MySQL)

3. Repository làm được những gì?

Chức năng Không dùng Repository Dùng Repository
Thêm mới Viết INSERT INTO... save(user)
Tìm theo ID Viết SELECT * FROM... WHERE id = ? findById(id)
Lấy tất cả Viết SELECT * FROM... findAll()
Cập nhật Viết UPDATE... SET... save(user)
Xóa Viết DELETE FROM... deleteById(id)
Đếm Viết SELECT COUNT(*) count()
Lợi ích:
  • Giảm code lặp lại
  • Tránh lỗi SQL
  • Tự động tối ưu query
  • Dễ bảo trì và kiểm thử

HỆ THỐNG REPOSITORY TRONG SPRING DATA JPA

1. Các interface Repository

Repository (Interface gốc)
↑
CrudRepository (Thêm CRUD cơ bản)
↑
PagingAndSortingRepository (Thêm phân trang, sắp xếp)
↑
JpaRepository (Thêm các phương thức JPA đặc thù) Nên dùng

2. So sánh các interface

Interface Phương thức Khi nào dùng
Repository Không có phương thức nào Đánh dấu, ít dùng trực tiếp
CrudRepository save(), findById(), findAll(), deleteById(), count() Chỉ cần CRUD cơ bản
PagingAndSortingRepository Thêm findAll(Pageable), findAll(Sort) Cần phân trang, sắp xếp
JpaRepository Thêm flush(), saveAndFlush(), deleteInBatch() Đầy đủ tính năng nhất (khuyến nghị)
Khuyến nghị: Nên dùng JpaRepository vì nó có đầy đủ phương thức nhất.

CÁC PHƯƠNG THỨC CÓ SẴN TRONG JPAREPOSITORY

1. Nhóm phương thức lưu (Save / Update)

Phương thức Mô tả Ví dụ
save(S entity) Lưu entity mới hoặc cập nhật entity đã có userRepository.save(user)
saveAll(Iterable<S> entities) Lưu nhiều entity cùng lúc userRepository.saveAll(listUsers)
saveAndFlush(S entity) Lưu và ngay lập tức đồng bộ xuống database userRepository.saveAndFlush(user)

2. Nhóm phương thức tìm kiếm (Find)

Phương thức Mô tả Ví dụ
findById(ID id) Tìm entity theo ID, trả về Optional Optional<User> user = userRepository.findById(1L)
findAll() Lấy tất cả entities List<User> users = userRepository.findAll()
findAllById(Iterable<ID> ids) Tìm nhiều entity theo danh sách ID userRepository.findAllById(List.of(1L, 2L, 3L))
getOne(ID id) Lấy tham chiếu (lazy loading) User user = userRepository.getOne(1L)

3. Nhóm phương thức kiểm tra (Check)

Phương thức Mô tả Ví dụ
existsById(ID id) Kiểm tra entity có tồn tại không boolean exists = userRepository.existsById(1L)
count() Đếm số lượng entities long count = userRepository.count()

4. Nhóm phương thức xóa (Delete)

Phương thức Mô tả Ví dụ
deleteById(ID id) Xóa entity theo ID userRepository.deleteById(1L)
delete(T entity) Xóa entity cụ thể userRepository.delete(user)
deleteAll() Xóa tất cả entities userRepository.deleteAll()
deleteAll(Iterable<? extends T> entities) Xóa nhiều entity userRepository.deleteAll(listUsers)
deleteAllInBatch() Xóa tất cả bằng 1 câu lệnh (nhanh hơn) userRepository.deleteAllInBatch()

package com.example.jpa;

import com.example.jpa.model.User;
import com.example.jpa.repository.UserRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.CommandLineRunner;
import org.springframework.stereotype.Component;

import java.util.List;
import java.util.Optional;

@Component // Spring tự động chạy khi khởi động
public class TestRepository implements CommandLineRunner {

    @Autowired
    private UserRepository userRepository;

    @Override
    public void run(String... args) throws Exception {
        System.out.println("===== BẮT ĐẦU TEST REPOSITORY =====");
        
        // 1. Thêm user mới
        testSave();
        
        // 2. Tìm tất cả users
        testFindAll();
        
        // 3. Tìm user theo ID
        testFindById();
        
        // 4. Cập nhật user
        testUpdate();
        
        // 5. Đếm số user
        testCount();
        
        // 6. Xóa user
        testDelete();
        
        System.out.println("===== KẾT THÚC TEST REPOSITORY =====");
    }

    private void testSave() {
        System.out.println("\n--- TEST 1: SAVE ---");
        
        // Tạo user mới
        User user1 = new User("Nguyen Van A", "a@gmail.com", "123456", 25);
        User user2 = new User("Tran Thi B", "b@gmail.com", "789012", 30);
        
        // Lưu vào database
        User savedUser1 = userRepository.save(user1);
        User savedUser2 = userRepository.save(user2);
        
        System.out.println("Đã lưu user1: ID = " + savedUser1.getId());
        System.out.println("Đã lưu user2: ID = " + savedUser2.getId());
    }

    private void testFindAll() {
        System.out.println("\n--- TEST 2: FIND ALL ---");
        
        List<User> users = userRepository.findAll();
        System.out.println("Tổng số users: " + users.size());
        
        for (User user : users) {
            System.out.println(" - " + user.getId() + ": " + user.getName() + " - " + user.getEmail());
        }
    }

    private void testFindById() {
        System.out.println("\n--- TEST 3: FIND BY ID ---");
        
        // Tìm user có ID = 1
        Optional<User> optional = userRepository.findById(1L);
        
        if (optional.isPresent()) {
            User user = optional.get();
            System.out.println("Tìm thấy: " + user.getName() + " - " + user.getEmail());
        } else {
            System.out.println("Không tìm thấy user ID = 1");
        }
        
        // Kiểm tra tồn tại
        boolean exists = userRepository.existsById(1L);
        System.out.println("User ID=1 tồn tại? " + exists);
    }

    private void testUpdate() {
        System.out.println("\n--- TEST 4: UPDATE ---");
        
        // Tìm user cần cập nhật
        Optional<User> optional = userRepository.findById(1L);
        
        if (optional.isPresent()) {
            User user = optional.get();
            System.out.println("Trước khi cập nhật: " + user.getName());
            
            // Cập nhật thông tin
            user.setName("Nguyen Van A (Updated)");
            user.setAge(26);
            
            // Lưu lại (save với ID đã có sẽ tự động update)
            User updatedUser = userRepository.save(user);
            System.out.println("Sau khi cập nhật: " + updatedUser.getName());
        }
    }

    private void testCount() {
        System.out.println("\n--- TEST 5: COUNT ---");
        
        long count = userRepository.count();
        System.out.println("Tổng số users hiện tại: " + count);
    }

    private void testDelete() {
        System.out.println("\n--- TEST 6: DELETE ---");
        
        // Đếm trước khi xóa
        long beforeCount = userRepository.count();
        System.out.println("Số user trước khi xóa: " + beforeCount);
        
        // Xóa user ID = 2
        userRepository.deleteById(2L);
        System.out.println("Đã xóa user ID = 2");
        
        // Đếm sau khi xóa
        long afterCount = userRepository.count();
        System.out.println("Số user sau khi xóa: " + afterCount);
    }
}

DERIVED QUERY METHODS

1. Định nghĩa

Derived Query Methods là các phương thức trong Repository mà Spring Data JPA tự động tạo câu truy vấn dựa trên tên phương thức.

2. Cú pháp đặt tên

Cú pháp
findBy + [Thuộc tính] + [Điều kiện] + [Kết hợp]

3. Các từ khóa trong Derived Query Methods

Từ khóa Ví dụ JPQL tương ứng
findBy... findByName WHERE name = ?1
findBy...And... findByNameAndEmail WHERE name = ?1 AND email = ?2
findBy...Or... findByNameOrEmail WHERE name = ?1 OR email = ?2
findBy...Containing findByNameContaining WHERE name LIKE %?1%
findBy...StartingWith findByNameStartingWith WHERE name LIKE ?1%
findBy...EndingWith findByNameEndingWith WHERE name LIKE %?1
findBy...IgnoreCase findByNameIgnoreCase WHERE UPPER(name) = UPPER(?1)
findBy...GreaterThan findByAgeGreaterThan WHERE age > ?1
findBy...LessThan findByAgeLessThan WHERE age < ?1
findBy...Between findByAgeBetween WHERE age BETWEEN ?1 AND ?2
findBy...IsNull findByPhoneNumberIsNull WHERE phone_number IS NULL
findBy...IsNotNull findByPhoneNumberIsNotNull WHERE phone_number IS NOT NULL
findBy...In findByAgeIn WHERE age IN (?1)
findBy...OrderBy... findByNameOrderByAgeDesc WHERE ... ORDER BY age DESC

4. Ví dụ thêm các phương thức vào UserRepository

UserRepository.java - Derived Query Methods
package com.example.jpa.repository;

import com.example.jpa.model.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // ===== 1. TÌM KIẾM CƠ BẢN =====
    
    // Tìm user theo email chính xác
    User findByEmail(String email);
    
    // Tìm user theo tên chính xác
    List<User> findByName(String name);
    
    // ===== 2. TÌM KIẾM VỚI NHIỀU ĐIỀU KIỆN =====
    
    // Tìm theo tên VÀ email
    User findByNameAndEmail(String name, String email);
    
    // Tìm theo tên HOẶC email
    List<User> findByNameOrEmail(String name, String email);
    
    // Tìm theo tên VÀ độ tuổi
    List<User> findByNameAndAge(String name, Integer age);
    
    // ===== 3. TÌM KIẾM VỚI ĐIỀU KIỆN SO SÁNH =====
    
    // Tìm user có độ tuổi lớn hơn
    List<User> findByAgeGreaterThan(Integer age);
    
    // Tìm user có độ tuổi nhỏ hơn
    List<User> findByAgeLessThan(Integer age);
    
    // Tìm user có độ tuổi trong khoảng
    List<User> findByAgeBetween(Integer startAge, Integer endAge);
    
    // Tìm user có độ tuổi lớn hơn hoặc bằng
    List<User> findByAgeGreaterThanEqual(Integer age);
    
    // ===== 4. TÌM KIẾM VỚI CHUỖI (LIKE) =====
    
    // Tìm user có tên chứa keyword (không phân biệt hoa thường)
    List<User> findByNameContainingIgnoreCase(String keyword);
    
    // Tìm user có tên bắt đầu bằng...
    List<User> findByNameStartingWith(String prefix);
    
    // Tìm user có tên kết thúc bằng...
    List<User> findByNameEndingWith(String suffix);
    
    // ===== 5. TÌM KIẾM VỚI NULL =====
    
    // Tìm user có số điện thoại null
    List<User> findByPhoneNumberIsNull();
    
    // Tìm user có số điện thoại không null
    List<User> findByPhoneNumberIsNotNull();
    
    // ===== 6. SẮP XẾP =====
    
    // Tìm tất cả sắp xếp theo tên tăng dần
    List<User> findAllByOrderByNameAsc();
    
    // Tìm tất cả sắp xếp theo tên giảm dần
    List<User> findAllByOrderByNameDesc();
    
    // Tìm theo độ tuổi và sắp xếp theo tên
    List<User> findByAgeOrderByNameAsc(Integer age);
    
    // ===== 7. KIỂM TRA TỒN TẠI =====
    
    // Kiểm tra email đã tồn tại chưa
    boolean existsByEmail(String email);
    
    // Kiểm tra có user nào theo độ tuổi không
    boolean existsByAge(Integer age);
    
    // ===== 8. ĐẾM =====
    
    // Đếm số user theo tên
    long countByName(String name);
    
    // Đếm số user theo độ tuổi
    long countByAge(Integer age);
    
    // ===== 9. XÓA =====
    
    // Xóa user theo email
    void deleteByEmail(String email);
    
    // Xóa user theo tên
    void deleteByName(String name);
}

@QUERY - TỰ VIẾT CÂU TRUY VẤN

1. Định nghĩa

Khi Derived Query Methods không đáp ứng được, chúng ta có thể tự viết câu truy vấn bằng @Query.

2. Cú pháp

@Query Syntax
@Query("JPQL statement")
List<Entity> methodName(@Param("param") Type param);

3. JPQL là gì?

JPQL (Java Persistence Query Language) là ngôn ngữ truy vấn tương tự SQL nhưng thao tác trên Entity (object) thay vì bảng.

4. So sánh SQL và JPQL

SQL JPQL
SELECT * FROM users SELECT u FROM User u
WHERE name = 'A' WHERE u.name = ?1
SELECT name, email FROM users SELECT u.name, u.email FROM User u
Thao tác trên bảng Thao tác trên Entity
Phụ thuộc database Độc lập database

5. Ví dụ thêm các phương thức với @Query

UserRepository.java - @Query Examples
package com.example.jpa.repository;

import com.example.jpa.model.User;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;

import java.util.List;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // ===== 1. JPQL - TRUY VẤN CƠ BẢN =====
    
    // Tìm user theo email (dùng tham số vị trí ?1)
    @Query("SELECT u FROM User u WHERE u.email = ?1")
    User findUserByEmail(String email);
    
    // Tìm user theo tên (dùng tham số vị trí)
    @Query("SELECT u FROM User u WHERE u.name = ?1")
    List<User> findUsersByName(String name);
    
    // ===== 2. JPQL - NHIỀU THAM SỐ =====
    
    // Tìm theo tên và email (thứ tự tham số)
    @Query("SELECT u FROM User u WHERE u.name = ?1 AND u.email = ?2")
    User findByNameAndEmail(String name, String email);
    
    // ===== 3. JPQL - NAMED PARAMETER (dùng @Param) =====
    
    // Dùng tên tham số cho dễ đọc
    @Query("SELECT u FROM User u WHERE u.name = :name AND u.age > :age")
    List<User> findByNameAndAgeGreaterThan(
        @Param("name") String name,
        @Param("age") Integer age);
    
    // ===== 4. JPQL - LIKE =====
    
    // Tìm tên chứa keyword
    @Query("SELECT u FROM User u WHERE u.name LIKE %:keyword%")
    List<User> searchByNameLike(@Param("keyword") String keyword);
    
    // ===== 5. JPQL - SẮP XẾP =====
    
    // Sắp xếp theo tên giảm dần
    @Query("SELECT u FROM User u WHERE u.age > :age ORDER BY u.name DESC")
    List<User> findUsersOlderThanOrderByNameDesc(@Param("age") Integer age);
    
    // ===== 6. JPQL - CHỌN MỘT PHẦN DỮ LIỆU =====
    
    // Chỉ lấy tên và email
    @Query("SELECT u.name, u.email FROM User u WHERE u.age > :age")
    List<Object[]> findNameAndEmailByAgeGreaterThan(@Param("age") Integer age);
    
    // ===== 7. JPQL - HÀM TỔNG HỢP =====
    
    // Tính tuổi trung bình
    @Query("SELECT AVG(u.age) FROM User u")
    Double getAverageAge();
    
    // Đếm số user theo tên
    @Query("SELECT u.name, COUNT(u) FROM User u GROUP BY u.name")
    List<Object[]> countUsersByName();
    
    // ===== 8. NATIVE QUERY - SQL THUẦN =====
    
    // Dùng SQL thuần (nativeQuery = true)
    @Query(value = "SELECT * FROM users WHERE age > :age", nativeQuery = true)
    List<User> findUsersByAgeNative(@Param("age") Integer age);
    
    // Native query với tên cột thật
    @Query(value = "SELECT full_name, email FROM users WHERE age BETWEEN :min AND :max",
           nativeQuery = true)
    List<Object[]> findNameAndEmailByAgeRangeNative(
        @Param("min") Integer minAge,
        @Param("max") Integer maxAge);
}

GIẢI THÍCH CHI TIẾT VỀ @QUERY

1. Tham số vị trí (?1, ?2)

Positional Parameters
@Query("SELECT u FROM User u WHERE u.name = ?1 AND u.email = ?2")
User findByNameAndEmail(String name, String email);
// ?1 = tham số thứ nhất (name)
// ?2 = tham số thứ hai (email)

2. Named Parameter (@Param)

Named Parameters
@Query("SELECT u FROM User u WHERE u.name = :name AND u.age > :age")
List<User> findByNameAndAgeGreaterThan(
    @Param("name") String name,
    @Param("age") Integer age);
// :name = tham số có tên "name"
// :age = tham số có tên "age"

3. Native Query

Native Query
@Query(value = "SELECT * FROM users WHERE age > ?1", nativeQuery = true)
List<User> findUsersByAgeNative(Integer age);
// nativeQuery = true: dùng SQL thuần, không phải JPQL
// "users" là tên bảng thật trong database
Lưu ý: Native Query phụ thuộc vào database cụ thể. Nếu đổi database (ví dụ từ MySQL sang PostgreSQL), có thể phải sửa query.

@MODIFYING VÀ @TRANSACTIONAL

1. Vấn đề với UPDATE/DELETE

Khi dùng @Query cho các câu lệnh UPDATE hoặc DELETE, cần thêm @Modifying và @Transactional.

2. Cú pháp

@Modifying & @Transactional
@Transactional // Phải có để thực hiện transaction
@Modifying // Báo đây là câu lệnh UPDATE/DELETE
@Query("UPDATE User u SET u.email = :email WHERE u.id = :id")
int updateEmail(@Param("id") Long id, @Param("email") String email);

3. Ví dụ

UserRepository.java - Modifying Queries
import org.springframework.transaction.annotation.Transactional;
import org.springframework.data.jpa.repository.Modifying;

// ===== 9. UPDATE VỚI @QUERY =====

// Cập nhật email (cần thêm @Modifying và @Transactional)
@Transactional
@Modifying
@Query("UPDATE User u SET u.email = :newEmail WHERE u.id = :userId")
int updateUserEmail(@Param("userId") Long userId, @Param("newEmail") String newEmail);

// ===== 10. DELETE VỚI @QUERY =====

// Xóa user theo email (cần @Modifying)
@Transactional
@Modifying
@Query("DELETE FROM User u WHERE u.email = :email")
int deleteUserByEmail(@Param("email") String email);

// Cập nhật tuổi theo tên
@Transactional
@Modifying
@Query("UPDATE User u SET u.age = :age WHERE u.name = :name")
int updateAgeByName(@Param("name") String name, @Param("age") Integer age);
Giải thích:
  • @Transactional: Đảm bảo các thao tác được thực hiện trong một transaction (hoặc tất cả thành công, hoặc tất cả rollback)
  • @Modifying: Báo cho Hibernate biết đây là câu lệnh UPDATE/DELETE, không phải SELECT
  • int: Giá trị trả về là số bản ghi bị ảnh hưởng

PHÂN TRANG VÀ SẮP XẾP VỚI PAGEABLE

1. Vấn đề

Khi có nhiều dữ liệu, không thể trả về tất cả trong một lần. Cần phân trang.

2. Sử dụng Pageable

UserRepository.java - Pagination
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;

public interface UserRepository extends JpaRepository<User, Long> {

    // Phân trang cho findAll
    Page<User> findAll(Pageable pageable);
    
    // Phân trang cho truy vấn có điều kiện
    Page<User> findByAgeGreaterThan(Integer age, Pageable pageable);
    
    // Phân trang với @Query
    @Query("SELECT u FROM User u WHERE u.name LIKE %:keyword%")
    Page<User> searchByName(@Param("keyword") String keyword, Pageable pageable);
}

3. Ví dụ sử dụng trong Service

UserService.java
import org.springframework.data.domain.Page;
import org.springframework.data.domain.PageRequest;
import org.springframework.data.domain.Pageable;
import org.springframework.data.domain.Sort;
import org.springframework.stereotype.Service;

@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;

    public Page<User> getUsersPage(int page, int size) {
        // Tạo Pageable: trang bắt đầu từ 0, mỗi trang 'size' bản ghi
        Pageable pageable = PageRequest.of(page, size);
        return userRepository.findAll(pageable);
    }

    public Page<User> getUsersByAgeSorted(int age, int page, int size, String sortBy) {
        // Sắp xếp theo trường 'sortBy' tăng dần
        Sort sort = Sort.by(sortBy).ascending();
        Pageable pageable = PageRequest.of(page, size, sort);
        return userRepository.findByAgeGreaterThan(age, pageable);
    }
}

4. Sử dụng Sort riêng

Sort Examples
// Sắp xếp không phân trang
List<User> findAll(Sort sort);

// Ví dụ
List<User> users = userRepository.findAll(Sort.by("name").ascending());
List<User> users = userRepository.findAll(Sort.by("age").descending());
List<User> users = userRepository.findAll(
    Sort.by("name").ascending().and(Sort.by("age").descending())
);

VÍ DỤ REPOSITORY HOÀN CHỈNH

UserRepository.java - Complete Example
package com.example.jpa.repository;

import com.example.jpa.model.User;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.data.jpa.repository.Modifying;
import org.springframework.data.jpa.repository.Query;
import org.springframework.data.repository.query.Param;
import org.springframework.stereotype.Repository;
import org.springframework.transaction.annotation.Transactional;

import java.util.List;
import java.util.Optional;

@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    // ===== 1. DERIVED QUERY METHODS =====
    
    // Tìm theo email
    Optional<User> findByEmail(String email);
    
    // Tìm theo tên chứa keyword
    List<User> findByNameContainingIgnoreCase(String name);
    
    // Tìm theo độ tuổi
    List<User> findByAge(Integer age);
    
    // Tìm theo khoảng tuổi
    List<User> findByAgeBetween(Integer startAge, Integer endAge);
    
    // Kiểm tra email tồn tại
    boolean existsByEmail(String email);
    
    // Đếm theo tên
    long countByName(String name);
    
    // ===== 2. JPQL QUERIES =====
    
    // Tìm user có tuổi lớn hơn
    @Query("SELECT u FROM User u WHERE u.age > :age")
    List<User> findUsersOlderThan(@Param("age") Integer age);
    
    // Tìm user có email kết thúc bằng domain
    @Query("SELECT u FROM User u WHERE u.email LIKE %:domain")
    List<User> findByEmailDomain(@Param("domain") String domain);
    
    // Tìm user và sắp xếp
    @Query("SELECT u FROM User u WHERE u.age BETWEEN :min AND :max ORDER BY u.name ASC")
    List<User> findByAgeRangeOrderByName(
        @Param("min") Integer minAge,
        @Param("max") Integer maxAge);
    
    // ===== 3. NATIVE QUERIES =====
    
    @Query(value = "SELECT * FROM users WHERE age > ?1", nativeQuery = true)
    List<User> findAdultUsers(Integer age);
    
    @Query(value = "SELECT COUNT(*) FROM users WHERE email LIKE ?1", nativeQuery = true)
    int countUsersByEmailDomain(String domain);
    
    // ===== 4. PAGINATION =====
    
    Page<User> findByAgeGreaterThan(Integer age, Pageable pageable);
    
    @Query("SELECT u FROM User u WHERE u.name LIKE %:keyword%")
    Page<User> searchUsers(@Param("keyword") String keyword, Pageable pageable);
    
    // ===== 5. MODIFYING QUERIES =====
    
    @Transactional
    @Modifying
    @Query("UPDATE User u SET u.age = :age WHERE u.email = :email")
    int updateAgeByEmail(@Param("email") String email, @Param("age") Integer age);
    
    @Transactional
    @Modifying
    @Query("DELETE FROM User u WHERE u.age < :age")
    int deleteUsersYoungerThan(@Param("age") Integer age);
    
    // ===== 6. AGGREGATION =====
    
    @Query("SELECT AVG(u.age) FROM User u")
    Double getAverageAge();
    
    @Query("SELECT MAX(u.age) FROM User u")
    Integer getMaxAge();
    
    @Query("SELECT MIN(u.age) FROM User u")
    Integer getMinAge();
}

TỔNG KẾT VỀ REPOSITORY

1. Ba cách để tạo truy vấn

Cách Ví dụ Khi nào dùng
Phương thức có sẵn findById(), save() CRUD cơ bản
Derived Query Methods findByNameAndEmail() Truy vấn đơn giản
@Query @Query("SELECT u FROM User u...") Truy vấn phức tạp

2. Các bước sử dụng Repository

  1. Tạo interface extends JpaRepository<T, ID>
  2. Khai báo các phương thức cần thiết
  3. @Autowired Repository vào Service
  4. Gọi các phương thức của Repository

3. Lưu ý quan trọng

  • Repository là interface, không phải class - Spring Data JPA tự động tạo implementation
  • Tên phương thức phải tuân theo quy tắc - Để Derived Query Methods hoạt động
  • @Transactional cho các thao tác sửa/xóa - Khi dùng @Modifying
  • Optional cho findById - Tránh NullPointerException
