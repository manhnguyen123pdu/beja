TẠO MODEL USER

1. Model là gì?

Model là class đại diện cho một đối tượng trong thực tế (User, Product, Employee...). Nó định nghĩa cấu trúc dữ liệu mà ứng dụng sẽ sử dụng.

Ví dụ:

  • Người dùng thực tế: Nguyễn Văn A, 25 tuổi, email a@gmail.com
  • Trong code: User object có các thuộc tính: id, name, age, email

2. Tạo package model

  1. Bước 1: Trong IntelliJ, mở project đã tạo ở Bài 1
  2. Bước 2: Tìm đến thư mục src/main/java/com/example/demo/
  3. Bước 3: Click chuột phải vào package com.example.demo → New → Package
  4. Bước 4: Nhập tên package: model → Enter

3. Tạo class User

File: src/main/java/com/example/demo/model/User.java

User.java
package com.example.demo.model;

/**
* Lớp User đại diện cho người dùng trong hệ thống
*/
public class User {

    // 1. Thuộc tính (private - tính đóng gói)
    private String id; // Mã người dùng
    private String name; // Họ tên
    private String email; // Email
    private String password; // Mật khẩu

    // 2. Constructor không tham số (BẮT BUỘC - Spring cần để tạo object từ JSON)
    public User() {
    }

    // 3. Constructor có tham số (tiện lợi khi tạo object)
    public User(String id, String name, String email, String password) {
        this.id = id;
        this.name = name;
        this.email = email;
        this.password = password;
    }

    // 4. Getter và Setter (BẮT BUỘC - để Jackson đọc/ghi dữ liệu)
    public String getId() {
        return id;
    }

    public void setId(String id) {
        this.id = id;
    }

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }

    public String getPassword() {
        return password;
    }

    public void setPassword(String password) {
        this.password = password;
    }
}

TẠO LIST LƯU TRỮ DỮ LIỆU TẠM

1. Vì sao cần List?

Trong thực tế, dữ liệu được lưu trong database. Nhưng để học, chúng ta dùng List để lưu tạm trong bộ nhớ.

2. Cập nhật UserController

File: src/main/java/com/example/demo/controller/UserController.java

UserController.java
package com.example.demo.controller;

import com.example.demo.model.User;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@RestController
@RequestMapping("/user")
public class UserController {

    // Tạo List để lưu trữ users (thay cho database)
    private List<User> users = new ArrayList<>();

    // Constructor: Khởi tạo dữ liệu mẫu
    public UserController() {
        // Thêm 2 user mẫu khi controller được khởi tạo
        users.add(new User("1", "Nguyen Van A", "a@gmail.com", "123456"));
        users.add(new User("2", "Tran Thi B", "b@gmail.com", "789012"));
        System.out.println("Đã tạo " + users.size() + " user mẫu");
    }
}

3. Giải thích

Dòng code Giải thích
private List<User> users = new ArrayList<>(); Tạo List để lưu các đối tượng User
public UserController() { ... } Constructor chạy khi controller được tạo, thêm dữ liệu mẫu
users.add(new User(...)) Thêm user mới vào List

@GETMAPPING - XỬ LÝ GET REQUEST

1. Định nghĩa

@GetMapping là annotation dùng để xử lý các request HTTP GET.

GET là phương thức dùng để lấy dữ liệu từ server.

2. Mục đích sử dụng

  • Lấy danh sách tất cả users
  • Lấy chi tiết một user theo ID
  • Tìm kiếm users theo điều kiện
  • Không làm thay đổi dữ liệu trên server

3. Cú pháp cơ bản

Cú pháp
@GetMapping("/duong-dan")
public KieuTraVe tenHam() {
    // Xử lý logic
    return duLieu;
}

4. Giải thích cú pháp

Thành phần Ý nghĩa Ví dụ
@GetMapping("/duong-dan") Định nghĩa URL mà client sẽ gọi @GetMapping("/getAll")
public Phạm vi truy cập - để Spring gọi được public
KieuTraVe Kiểu dữ liệu trả về List<User>, String, User
tenHam() Tên hàm xử lý getAllUsers()
return duLieu; Dữ liệu trả về cho client return users;

5. Ví dụ 1: GET trả về chuỗi đơn giản

Ví dụ
@GetMapping("/hello")
public String sayHello() {
    return "Hello from Spring Boot!";
}

Kết quả: http://localhost:8080/user/hello → "Hello from Spring Boot!"

6. Ví dụ 2: GET không tham số - Lấy danh sách

Ví dụ
@GetMapping("/getAll")
public List<User> getAllUsers() {
    return users; // Trả về danh sách users
}

Giải thích ví dụ:

  • @GetMapping("/getAll"): Xử lý GET request đến đường dẫn /user/getAll
  • public List<User>: Trả về List các đối tượng User
  • return users;: Trả danh sách users đã khởi tạo

Kết quả khi gọi http://localhost:8080/user/getAll:

JSON Output
[
    {
        "id": "1",
        "name": "Nguyen Van A",
        "email": "a@gmail.com",

        "id": "2",

7. Lưu ý quan trọng

Khi trả về object (User, List<User>), Spring Boot tự động chuyển thành JSON nhờ thư viện Jackson. Jackson sử dụng getter để đọc dữ liệu.

Nếu thiếu getter:

Thiếu getter
public class User {
    private String name; // Không có getName()
}
// Kết quả: {} - JSON rỗng

Nếu có đủ getter:

Có getter
public class User {
    private String name;
    public String getName() { return name; } // Bắt buộc phải có
}
// Kết quả: {"name": "Nguyen Van A"}

@REQUESTPARAM - LẤY THAM SỐ TỪ QUERY STRING

1. Định nghĩa

@RequestParam là annotation dùng để lấy tham số từ URL dạng query string (?key=value).

2. Mục đích sử dụng

  • Lấy tham số tìm kiếm: /user/search?name=Nguyen
  • Lấy tham số phân trang: /user/list?page=1&limit=10
  • Lấy tham số lọc: /user/filter?age=25&active=true

3. Cú pháp

Cú pháp
@GetMapping("/duong-dan")
public KieuTraVe tenHam(@RequestParam("ten-tham-so") KieuDuLieu tenBien) {
    // Xử lý với tenBien
    return duLieu;
}

4. Giải thích cú pháp

Thành phần Ý nghĩa Ví dụ
@RequestParam("ten-tham-so") Lấy giá trị từ tham số trên URL @RequestParam("name")
KieuDuLieu Kiểu dữ liệu của tham số String, int, boolean
tenBien Biến nhận giá trị String userName

5. Ví dụ 1: Một tham số

Ví dụ
@GetMapping("/search")
public String searchUser(@RequestParam("name") String name) {
    return "Đang tìm user có tên: " + name;
}

Cách gọi: http://localhost:8080/user/search?name=Nguyen

Kết quả: "Đang tìm user có tên: Nguyen"

6. Ví dụ 2: Nhiều tham số

Ví dụ
@GetMapping("/filter")
public String filterUsers(
        @RequestParam("page") int page,
        @RequestParam("limit") int limit) {
    return "Trang " + page + ", mỗi trang " + limit + " bản ghi";
}

Cách gọi: http://localhost:8080/user/filter?page=1&limit=10

Kết quả: "Trang 1, mỗi trang 10 bản ghi"

7. Ví dụ 3: Tham số không bắt buộc (required = false)

Ví dụ
@GetMapping("/search2")
public String searchUser2(
        @RequestParam(value = "name", required = false) String name) {
    if (name == null) {
        return "Vui lòng nhập tên cần tìm";
    }
    return "Đang tìm user có tên: " + name;
}

Cách gọi:

  • http://localhost:8080/user/search2?name=Nguyen → "Đang tìm user có tên: Nguyen"
  • http://localhost:8080/user/search2 → "Vui lòng nhập tên cần tìm"

Ví dụ
@GetMapping("/list")
public String getUserList(
        @RequestParam(value = "page", defaultValue = "1") int page,

    return "Trang " + page + ", hiển thị " + limit + " user";

Cách gọi:

  • http://localhost:8080/user/list → "Trang 1, hiển thị 5 user"

@PATHVARIABLE - LẤY THAM SỐ TỪ URL

1. Định nghĩa

@PathVariable là annotation dùng để lấy giá trị từ URL dạng đường dẫn động (/user/{id}).

2. Mục đích sử dụng

  • Lấy ID từ URL: /user/1 → lấy id = 1
  • Lấy thông tin chi tiết: /product/iphone-14
  • URL có cấu trúc: /user/1/order/5

3. Cú pháp

Cú pháp
@GetMapping("/duong-dan/{ten-bien}")
public KieuTraVe tenHam(@PathVariable("ten-bien") KieuDuLieu tenBien) {
    // Xử lý với tenBien
    return duLieu;
}

4. Giải thích cú pháp

Thành phần Ý nghĩa Ví dụ
{ten-bien} Định nghĩa biến trong URL {id}
@PathVariable("ten-bien") Lấy giá trị từ URL @PathVariable("id")
KieuDuLieu Kiểu dữ liệu int, String
tenBien Biến nhận giá trị int userId

5. Ví dụ 1: Một PathVariable

Ví dụ
@GetMapping("/getUser/{id}")
public String getUserById(@PathVariable("id") String id) {
    return "Lấy user có ID: " + id;

Cách gọi: http://localhost:8080/user/getUser/5

6. Ví dụ 2: Bỏ qua tên trong @PathVariable (nếu trùng tên biến)

        return "Không tìm thấy user ID: " + id;
    }
}

VÍ DỤ TỔNG HỢP - USER CONTROLLER HOÀN CHỈNH

1. File UserController.java hoàn chỉnh

UserController.java
package com.example.demo.controller;

import com.example.demo.model.User;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.List;

@RestController
@RequestMapping("/user")
public class UserController {

    // 1. Tạo List lưu trữ users
    private List<User> users = new ArrayList<>();

    // 2. Constructor khởi tạo dữ liệu mẫu
    public UserController() {
        users.add(new User("1", "Nguyen Van A", "a@gmail.com", "123456"));
        users.add(new User("2", "Tran Thi B", "b@gmail.com", "789012"));
        System.out.println("Đã khởi tạo " + users.size() + " user mẫu");
    }

    // 3. GET - Lấy tất cả users
    @GetMapping("/getAll")
    public List<User> getAllUsers() {
        return users;
    }

    // 4. GET - Lấy user theo ID (dùng PathVariable)
    @GetMapping("/getUser/{id}")
    public String getUserById(@PathVariable String id) {
        for (User user : users) {
            if (user.getId().equals(id)) {
                return "Tìm thấy: " + user.getName();
            }
        }
        return "Không tìm thấy user ID: " + id;
    }

    // 5. GET - Tìm kiếm user theo tên (dùng RequestParam)
    @GetMapping("/search")
    public String searchUser(@RequestParam String name) {
        List<String> foundUsers = new ArrayList<>();
        for (User user : users) {
            if (user.getName().toLowerCase().contains(name.toLowerCase())) {
                foundUsers.add(user.getName());
            }
        }

        if (foundUsers.isEmpty()) {
            return "Không tìm thấy user nào có tên chứa: " + name;
        }
        return "Tìm thấy: " + String.join(", ", foundUsers);
    }

    // 6. POST - Thêm user mới
    @PostMapping("/addUser")
    public String addUser(@RequestBody User user) {
        users.add(user);
        return "Đã thêm user: " + user.getName() + ", tổng số: " + users.size();
    }

    // 7. DELETE - Xóa user theo ID
    @DeleteMapping("/deleteUser/{id}")
    public String deleteUser(@PathVariable String id) {
        boolean removed = users.removeIf(user -> user.getId().equals(id));

        if (removed) {
            return "Đã xóa user ID: " + id + ", còn lại: " + users.size() + " user";
        } else {
            return "Không tìm thấy user ID: " + id;
        }
    }

    // 8. GET - Lấy tổng số user
    @GetMapping("/count")
    public String getCount() {
        return "Tổng số user: " + users.size();
    }
}

TỔNG KẾT BÀI HỌC

1. Những gì đã học

STT Kiến thức Mô tả
1 Model Class đại diện cho dữ liệu
2 @GetMapping Xử lý GET request - lấy dữ liệu
3 @RequestParam Lấy tham số từ query string ?key=value
4 @PathVariable Lấy tham số từ URL /{id}
5 @PostMapping Xử lý POST request - thêm dữ liệu
6 @RequestBody Nhận JSON từ client, chuyển thành object
7 @DeleteMapping Xử lý DELETE request - xóa dữ liệu
8 List Lưu trữ dữ liệu tạm thời

2. So sánh các annotation

Annotation Dùng cho Lấy dữ liệu từ Ví dụ URL
@RequestParam GET Query string /search?name=Nguyen
@PathVariable GET, DELETE URL path /user/1
@RequestBody POST, PUT Body JSON Gửi JSON trong body
