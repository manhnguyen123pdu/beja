NỘI DUNG 1: PHƯƠNG THỨC GET VÀ POST

1.1. Khái niệm

HTTP Methods (còn gọi là HTTP verbs) là các phương thức giao tiếp giữa client và server, cho biết hành động mà client muốn server thực hiện trên tài nguyên được chỉ định bởi URL.

Mỗi HTTP request đều có một method. Khi Servlet nhận được request, nó sẽ dựa vào method để gọi phương thức tương ứng: doGet(), doPost(), doPut(), doDelete()...

1.2 Các HTTP Methods phổ biến

Method Mục đích Idempotent?¹ Safe?² Dùng trong Servlet
GET Lấy dữ liệu từ server Có Có doGet()
POST Gửi dữ liệu để tạo mới Không Không doPost()
PUT Cập nhật toàn bộ tài nguyên Có Không doPut()
DELETE Xóa tài nguyên Có Không doDelete()
PATCH Cập nhật một phần tài nguyên Không Không Không hỗ trợ mặc định
HEAD Lấy header giống GET nhưng không có body Có Có doHead()
OPTIONS Hỏi server hỗ trợ method nào Có Có doOptions()

1 Idempotent: Gọi nhiều lần đều cho kết quả giống như gọi một lần (không gây tác dụng phụ thay đổi dữ liệu nhiều lần).

2 Safe: Không làm thay đổi dữ liệu trên server.

1.2. Phương thức GET

Định nghĩa

Dùng để lấy dữ liệu (retrieve)

Dữ liệu gửi kèm qua URL (query string)

Có giới hạn độ dài (thường 2048-4096 ký tự tùy browser)

Dữ liệu hiển thị trên thanh địa chỉ → không an toàn cho mật khẩu

Cú pháp GET request từ trình duyệt

GET /servlet-path?ten1=gia tri1&ten2=gia tri2 HTTP/1.1 Host: localhost:8080

Giải thích cú pháp:

GETTên phương thức
/servlet-pathĐường dẫn đến servlet
?Dấu bắt đầu query string (chuỗi tham số)
ten1=gia tri1Tham số thứ nhất (tên=giá trị)
&Dấu phân cách giữa các tham số
ten2=gia tri2Tham số thứ hai
HTTP/1.1Phiên bản giao thức
Host:Tên miền server

Cú pháp xử lý GET trong Servlet

ViDuGetServlet.java
@WebServlet("/vi-du-get") public class ViDuGetServlet extends HttpServlet { @Override protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException { // Xử lý GET request } }

Giải thích cú pháp:

1.3. Phương thức POST

Định nghĩa

Dùng để gửi dữ liệu lên server (tạo mới)

Dữ liệu nằm trong body của request → không hiển thị trên URL

Không giới hạn độ dài

An toàn hơn GET cho dữ liệu nhạy cảm

Không cache được, không bookmark được

Cú pháp POST request

POST /servlet-path HTTP/1.1 Host: localhost:8080 Content-Type: application/x-www-form-urlencoded Content-Length: 27 username=admin&password=123

Giải thích cú pháp:

POST /servlet-path HTTP/1.1Dòng request: phương thức POST, đường dẫn, giao thức
Host: localhost:8080Header: chỉ định server
Content-Type: application/...Header: kiểu dữ liệu gửi lên (form)
Content-Length: 27Header: độ dài của body (tính bằng byte)
(dòng trống)Phân cách giữa headers và body
username=admin&password=123Body: dữ liệu form gửi lên

Cú pháp xử lý POST trong Servlet

ViDuPostServlet.java
@WebServlet("/vi-du-post") public class ViDuPostServlet extends HttpServlet { @Override protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException { // Xử lý POST request } }

Giải thích: Tương tự doGet, nhưng phương thức là doPost để xử lý POST request.

1.4. Phương thức PUT

Định nghĩa

Dùng để cập nhật toàn bộ tài nguyên tại URL chỉ định

Thay thế hoàn toàn nội dung cũ bằng nội dung mới

Idempotent: gọi 10 lần cũng cho kết quả giống như gọi 1 lần

Thường dùng trong REST API để cập nhật đối tượng

Cú pháp PUT request

PUT /api/users/5 HTTP/1.1 Host: localhost:8080 Content-Type: application/json Content-Length: 45 { "username": "nguyenvana", "email": "ana@example.com" }

Giải thích cú pháp:

PUT /api/users/5 HTTP/1.1 Dòng request: phương thức PUT, đường dẫn (kèm id=5), giao thức
Host: localhost:8080 Header: chỉ định server
Content-Type: application/json Header: dữ liệu gửi lên là JSON (thường dùng với PUT)
Content-Length: 45 Header: độ dài của body (tính bằng byte)
(dòng trống) Phân cách giữa headers và body
{ "username": "...", "email": "... } Body: dữ liệu JSON dùng để cập nhật toàn bộ user

Lưu ý quan trọng về PUT

  • Nếu tài nguyên chưa tồn tại, một số server có thể tạo mới (theo đặc tả HTTP, PUT có thể tạo mới)
  • PUT yêu cầu gửi toàn bộ các trường của đối tượng, không chỉ trường thay đổi
  • Để cập nhật một phần, nên dùng PATCH thay vì PUT

Cú pháp xử lý PUT trong Servlet

ViDuPutServlet.java
@WebServlet("/api/users/*") public class ViDuPutServlet extends HttpServlet { @Override protected void doPut(HttpServletRequest request,HttpServletResponse response) throws ServletException, IOException { } }

Giải thích: Phương thức doPut được gọi khi request có method PUT. Cần đọc id từ URL và dữ liệu từ body request để cập nhật.

1.5. Phương thức DELETE

Định nghĩa

Dùng để xóa tài nguyên tại URL chỉ định

Idempotent: xóa lần 2 không còn gì để xóa (vẫn trả về thành công)

Thường không có body (hoặc body rỗng)

Thành công thường trả về mã 204 No Content (không có nội dung)

Cú pháp DELETE request

DELETE /api/users/5 HTTP/1.1 Host: localhost:8080 Content-Length: 0

Giải thích cú pháp:

DELETE /api/users/5 HTTP/1.1 Dòng request: phương thức DELETE, đường dẫn (chỉ rõ tài nguyên cần xóa), giao thức
Host: localhost:8080 Header: chỉ định server
Content-Length: 0 Header: không có body dữ liệu
(dòng trống) Kết thúc request, không có body

Lưu ý quan trọng về DELETE

  • DELETE không nên có body (theo đặc tả HTTP, nhưng một số API vẫn cho phép)
  • Nếu xóa thành công nhưng không có nội dung trả về, dùng status code 204 (No Content)
  • Nếu xóa thành công và muốn trả về thông báo, dùng status code 200 (OK)
  • Nếu không tìm thấy tài nguyên để xóa, trả về 404 (Not Found)

Cú pháp xử lý DELETE trong Servlet

ViDuDeleteServlet.java
@WebServlet("/api/users/*") public class ViDuDeleteServlet extends HttpServlet { @Override protected void doDelete(HttpServletRequest request,HtpServletResponse response) throws ServletException, IOException { } }

Giải thích: Phương thức doDelete được gọi khi request có method DELETE. Cần lấy id từ URL, thực hiện xóa và trả về status code phù hợp.

1.6. Tóm tắt các phương thức HTTP

Method Mục đích Idempotent Dữ liệu Method trong Servlet
GET Lấy dữ liệu Có URL (query string) doGet()
POST Tạo mới Không Body doPost()
PUT Cập nhật toàn bộ Có Body doPut()
DELETE Xóa Có Thường không có doDelete()

NỘI DUNG 2: NHẬN DỮ LIỆU TỪ FORM

2.1. Các phương thức đọc dữ liệu từ HttpServletRequest

Định nghĩa

HttpServletRequest là đối tượng đại diện cho request từ client gửi lên server.

Servlet container tạo đối tượng này và truyền vào các phương thức doGet(), doPost(), doPut(), doDelete() của Servlet.

Thông qua đối tượng này, Servlet có thể đọc được mọi thông tin từ client: tham số, header, cookie, session, dữ liệu upload,...

Các nhóm phương thức chính
Nhóm phương thức Mục đích
getParameter...() Đọc dữ liệu từ form hoặc query string
getHeader...() Đọc thông tin header HTTP
getCookie() Đọc cookie từ client
getSession() Làm việc với session
get...() Lấy thông tin về request (URL, IP, method,...)
getReader() / getInputStream() Đọc raw data từ body

2.1.1. getParameter(String name)

Cú pháp:
String giaTri = request.getParameter("tenThamSo");
Giải thích:
  • request: Đối tượng HttpServletRequest
  • getParameter(): Phương thức đọc tham số
  • "tenThamSo": Tên tham số cần đọc (phân biệt hoa-thường)
  • Trả về: String - giá trị của tham số, hoặc null nếu không tồn tại
Ví dụ:
// Với URL: /servlet?username=nguyenvana&age=20 String username = request.getParameter("username"); // = "nguyenvana" String age = request.getParameter("age"); // = "20" String email = request.getParameter("email"); // = null (không có)
Ví dụ với doGet

File demo.java (Servlet)

demo.java
package test; import java.io.IOException; import javax.servlet.ServletException; import javax.servlet.annotation.WebServlet; import javax.servlet.http.HttpServlet; import javax.servlet.http.HttpServletRequest; import javax.servlet.http.HttpServletResponse; @WebServlet("/hello") public class demo extends HttpServlet { @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { // 1. Lấy tham số username từ URL (GET) String username = req.getParameter("username"); // 2. Set attribute để chuyển sang JSP req.setAttribute("username", username); // 3. Forward sang login.jsp req.getRequestDispatcher("/login.jsp").forward(req, resp); } }

File login.jsp

login.jsp
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%> <!DOCTYPE html> <html> <head> <meta charset="UTF-8"> <title>Insert title here</title> </head> <body> <form action="hello" method="get"> <input type="text" name="username" placeholder="Tên đăng nhập"> <input type="password" name="password" placeholder="Mật khẩu"> <h1>Hello ${username}</h1> <button type="submit">Đăng nhập</button> </form> </body> </html>
Cách hoạt động hiện tại:
  1. User nhập username và password
  2. Submit form (method="get") → dữ liệu hiện trên URL
  3. Servlet nhận được username (nhưng không dùng password)
  4. Forward sang JSP và hiển thị "Hello [username]"
Ví dụ với doPost
Demo.java
package test; import java.io.IOException; import javax.servlet.ServletException; import javax.servlet.annotation.WebServlet; import javax.servlet.http.HttpServlet; import javax.servlet.http.HttpServletRequest; import javax.servlet.http.HttpServletResponse; @WebServlet("/hello") public class Demo extends HttpServlet { // GET = hiển thị form @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { req.getRequestDispatcher("/login.jsp").forward(req, resp); } // POST = xử lý dữ liệu form @Override protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String username = req.getParameter("username"); req.setAttribute("username", username); req.getRequestDispatcher("/login.jsp").forward(req, resp); } }

2.1.2. getParameterValues(String name) - Đọc nhiều giá trị (checkbox)

Cú pháp:
String[] cacGiaTri = request.getParameterValues("tenThamSo");
Giải thích:
  • Dùng cho các tham số có nhiều giá trị (checkbox, select multiple)
  • Trả về mảng String[] chứa tất cả giá trị
  • Trả về null nếu tham số không tồn tại
Ví dụ:
// HTML: <input type="checkbox" name="hobby" value="reading"> Đọc sách // <input type="checkbox" name="hobby" value="travel"> Du lịch // Nếu người dùng chọn cả 2: String[] hobbies = request.getParameterValues("hobby"); // hobbies = ["reading", "travel"] // Nếu không chọn gì: hobbies = null

2.2. Xử lý encoding tiếng Việt

Cú pháp set encoding cho request (POST)
// PHẢI đặt TRƯỚC khi đọc bất kỳ parameter nào request.setCharacterEncoding("UTF-8");
Giải thích:
  • setCharacterEncoding: Thiết lập bảng mã cho request
  • "UTF-8": Bảng mã Unicode hỗ trợ tiếng Việt
  • Chỉ có tác dụng với POST request
  • Phải gọi TRƯỚC khi đọc parameter lần đầu tiên
Cú pháp set encoding cho response
response.setContentType("text/html;charset=UTF-8");
Giải thích:
  • setContentType: Thiết lập kiểu nội dung trả về
  • "text/html": Nội dung là HTML
  • charset=UTF-8: Sử dụng bảng mã UTF-8
  • Đảm bảo tiếng Việt hiển thị đúng trên trình duyệt

NỘI DUNG 4: HTTPSERVLETRESPONSE - CHI TIẾT CÚ PHÁP

4.1. Thiết lập response

4.1.1. setContentType()

response.setContentType("text/html;charset=UTF-8");

Giải thích: Thiết lập kiểu nội dung (MIME type) của response

  • text/html: Nội dung là HTML
  • text/plain: Văn bản thuần túy
  • application/json: Dữ liệu JSON
  • application/xml: Dữ liệu XML
  • image/jpeg: Ảnh JPEG
  • charset=UTF-8: Bảng mã UTF-8 cho tiếng Việt

4.1.2. setCharacterEncoding()

response.setCharacterEncoding("UTF-8");

Giải thích: Chỉ thiết lập encoding, không thiết lập content type. Nên dùng setContentType("text/html;charset=UTF-8") cho gọn.

4.2. Gửi dữ liệu về client

4.2.1. getWriter() - Ghi text

PrintWriter out = response.getWriter(); out.println("Nội dung cần ghi"); out.close(); // Đóng writer (không bắt buộc nhưng nên làm)
Giải thích:
  • getWriter(): Lấy đối tượng PrintWriter để ghi text
  • println(): Ghi một dòng văn bản (tự động xuống dòng)
  • print(): Ghi văn bản (không xuống dòng)
  • close(): Đóng writer, giải phóng tài nguyên
Ví dụ:
PrintWriter out = response.getWriter(); out.println("<!DOCTYPE html>"); out.println("<html>"); out.println("<head><title>Trang web</title></head>"); out.println("<body>"); out.println("<h1>Xin chào</h1>"); out.println("</body>"); out.println("</html>");

Ví dụ tạo trang HTML hoàn chỉnh

@WebServlet("/tao-trang") public class TaoTrangServlet extends HttpServlet { @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { // Thiết lập content type resp.setContentType("text/html;charset=UTF-8"); // Lấy PrintWriter PrintWriter out = resp.getWriter(); // Ghi toàn bộ trang HTML out.println("<!DOCTYPE html>"); out.println("<html lang='vi'>"); out.println("<head>"); out.println(" <meta charset='UTF-8'>"); out.println(" <meta name='viewport' content='width=device-width, initial-scale=1.0'>"); out.println(" <title>Trang được tạo từ Servlet</title>"); out.println(" <style>"); out.println(" body { font-family: Arial, sans-serif; margin: 50px; }"); out.println(" h1 { color: blue; }"); out.println(" p { color: gray; }"); out.println(" </style>"); out.println("</head>"); out.println("<body>"); out.println(" <h1>Chào mừng bạn!</h1>"); out.println(" <p>Đây là trang HTML được tạo hoàn toàn từ Servlet.</p>"); out.println(" <p>Thời gian hiện tại: " + new java.util.Date() + "</p>"); out.println("</body>"); out.println("</html>"); // Đóng writer out.close(); } }

NỘI DUNG 6: REDIRECT VÀ FORWARD

6.1. Redirect (Chuyển hướng)

Định nghĩa

Redirect là kỹ thuật yêu cầu trình duyệt chuyển sang một URL khác. Server gửi response với mã 302 và header Location.

Cú pháp

// Chuyển hướng đến URL tuyệt đối response.sendRedirect("https://google.com"); // Chuyển hướng trong cùng ứng dụng (relative) response.sendRedirect("welcome"); // Chuyển hướng với context path response.sendRedirect(request.getContextPath() + "/welcome");
Giải thích:
  • sendRedirect(String location): Phương thức chuyển hướng
  • location: Đường dẫn đích (có thể tuyệt đối hoặc tương đối)
  • request.getContextPath(): Lấy tên ứng dụng (ví dụ: /FirstServletProject)

Ví dụ

DangNhapServlet.java
@WebServlet("/dang-nhap") public class DangNhapServlet extends HttpServlet { @Override protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String username = req.getParameter("username"); String password = req.getParameter("password"); if ("admin".equals(username) && "123".equals(password)) { // Đăng nhập thành công - chuyển đến trang chào mừng // CÚ PHÁP: sendRedirect - client sẽ gửi request mới đến /welcome resp.sendRedirect("welcome"); } else { // Đăng nhập thất bại - quay lại trang login // CÚ PHÁP: sendRedirect với tham số báo lỗi resp.sendRedirect("login.html?error=1"); } } }

6.2. Forward (Chuyển tiếp)

Định nghĩa

Forward là kỹ thuật chuyển tiếp request từ servlet này sang servlet khác trong cùng server. Trình duyệt không biết có sự chuyển tiếp.

Cú pháp

// Lấy RequestDispatcher RequestDispatcher dispatcher = request.getRequestDispatcher("dich"); // Forward request và response dispatcher.forward(request, response);

Hoặc viết gọn:

request.getRequestDispatcher("dich").forward(request, response);
Giải thích:
  • getRequestDispatcher(String path): Lấy đối tượng chuyển tiếp
  • path: Đường dẫn đích (có thể là servlet hoặc JSP)
  • forward(request, response): Thực hiện chuyển tiếp

Ví dụ

XuLyServlet.java
@WebServlet("/xu-ly") public class XuLyServlet extends HttpServlet { @Override protected void doPost(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { String username = req.getParameter("username"); if (username == null || username.isEmpty()) { // Lỗi - forward về form đăng ký // CÚ PHÁP: setAttribute để gửi thông báo lỗi req.setAttribute("error", "Tên không được để trống"); // CÚ PHÁP: forward - request được chuyển đến /show-form request.getRequestDispatcher("/show-form").forward(req, resp); } else { // Thành công - forward đến servlet hiển thị req.setAttribute("username", username); request.getRequestDispatcher("/hien-thi").forward(req, resp); } } }

6.4. RequestDispatcher - Các phương thức

6.4.1. forward()

dispatcher.forward(request, response);

Giải thích: Chuyển tiếp request và response đến tài nguyên khác. Không được ghi response trước khi forward (sẽ có lỗi).

6.4.2. include()

dispatcher.include(request, response);

Giải thích: Bao gồm nội dung của tài nguyên khác vào response hiện tại. Giống "include" trong JSP. Có thể include nhiều tài nguyên.

Ví dụ include:

TrangChuServlet.java
@WebServlet("/trang-chu") public class TrangChuServlet extends HttpServlet { @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { resp.setContentType("text/html;charset=UTF-8"); PrintWriter out = resp.getWriter(); out.println("<html><body>"); out.println("<h1>Đầu trang</h1>"); // Include header request.getRequestDispatcher("/header").include(req, resp); out.println("<p>Nội dung chính</p>"); // Include footer request.getRequestDispatcher("/footer").include(req, resp); out.println("</body></html>"); } }
