PHẦN 1: TỔNG QUAN VỀ API VÀ REST

1.1 API là gì? (Lý thuyết)

Định nghĩa API (Application Programming Interface) là một tập hợp các quy tắc và giao thức cho phép các ứng dụng phần mềm giao tiếp với nhau.

Ví dụ thực tế

Hãy tưởng tượng bạn vào một nhà hàng:

Thành phần Vai trò Trong API
Bạn Người dùng Client (App, Web)
Người phục vụ Người trung gian API
Đầu bếp Người xử lý Server
Thực đơn Danh sách các món có thể gọi Documentation
Món ăn Dữ liệu Response

Quy trình: Bạn xem thực đơn → API cung cấp danh sách các endpoint
Bạn gọi món (gửi request) → Client gọi API
Người phục vụ chuyển cho đầu bếp → API chuyển cho Server
Đầu bếp nấu món → Server xử lý
Món ăn được mang ra → Server trả response

Tại sao cần API?

Vấn đề Giải pháp từ API
Ứng dụng cần lấy dữ liệu từ server API cung cấp dữ liệu qua HTTP
Nhiều nền tảng (Web, iOS, Android) cùng dùng chung dữ liệu API là lớp trung gian duy nhất
Bảo mật dữ liệu API kiểm soát ai được truy cập gì
Tách biệt frontend và backend Frontend chỉ gọi API, không cần biết backend xử lý thế nào

1.2 REST API là gì? (Lý thuyết)

Định nghĩa REST (Representational State Transfer) là một phong cách kiến trúc để xây dựng API, do Roy Fielding đề xuất năm 2000.

6 Nguyên tắc của REST

Nguyên tắc Giải thích Ví dụ
1. Client-Server Tách biệt giao diện và xử lý Frontend (React) gọi Backend (Servlet)
2. Stateless Server không lưu trạng thái client Mỗi request phải có đầy đủ thông tin
3. Cacheable Response có thể được lưu lại Các request GET có thể cache
4. Uniform Interface Giao diện thống nhất Dùng chung cách đặt URL, method HTTP
5. Layered System Hệ thống phân lớp Có thể thêm proxy, gateway ở giữa
6. Code on Demand Server gửi code xuống client (Tùy chọn, ít dùng)

Stateless - Quan trọng nhất!

- Stateful (CÓ lưu trạng thái) - KHÔNG phải REST:
Request 1: "Tôi là A, cho tôi xem sản phẩm"
Server: Nhớ "A đang xem sản phẩm"

Request 2: "Thêm vào giỏ"
Server: Nhớ "A thêm sản phẩm X vào giỏ"

Request 3: "Thanh toán"
Server: Nhớ "A thanh toán giỏ hàng"

+ Stateless (KHÔNG lưu trạng thái) - REST đúng chuẩn:
Request 1: "Tôi là A, token=xyz123, cho tôi xem sản phẩm"
Request 2: "Tôi là A, token=xyz123, thêm sản phẩm X vào giỏ hàng của tôi"
Request 3: "Tôi là A, token=xyz123, thanh toán giỏ hàng của tôi"

REST API trông như thế nào?

GET    /users          → Lấy danh sách users
GET    /users/5        → Lấy user có ID = 5
POST   /users          → Thêm user mới
PUT    /users/5        → Cập nhật user ID = 5
DELETE /users/5        → Xóa user ID = 5

1.3 HTTP Method (Phương thức) (Lý thuyết)

Method Mục đích Idempotent? Có body? An toàn?
GET Lấy dữ liệu Có Không Có
POST Tạo mới Không Có Không

Giải thích chi tiết

HTTP Methods trong REST API

GET

1. GET - Lấy dữ liệu

GET /users         → Lấy tất cả users
GET /users/1       → Lấy user có id = 1
GET /users?name=An → Tìm user tên "An"

Đặc điểm:
- Không làm thay đổi dữ liệu trên server.
- Có thể lưu cache.
- Dữ liệu gửi qua URL (Query String).

POST

2. POST - Tạo mới

POST /users

Body:
{
  "name": "Nguyễn Văn A",
  "email": "a@gmail.com"
}

Đặc điểm:
- Tạo tài nguyên mới.
- Gửi 2 lần → tạo 2 bản ghi khác nhau.
- Dữ liệu gửi trong Body.

PUT

3. PUT - Cập nhật toàn bộ

PUT /users/1

Body:
{
  "name": "Nguyễn Văn A (sửa)",
  "email": "a_new@gmail.com"
}

Đặc điểm:
- Thay thế hoàn toàn tài nguyên.
- Gửi 2 lần → kết quả giống nhau.
- Phải gửi đầy đủ thông tin.

PATCH

4. PATCH - Cập nhật một phần

PATCH /users/1

Body:
{
  "name": "Nguyễn Văn A (sửa)"
}

Đặc điểm:
- Chỉ cập nhật các trường được gửi.
- Tiết kiệm băng thông.

DELETE

5. DELETE - Xóa

DELETE /users/1

Đặc điểm:
- Xóa tài nguyên.
- Gửi 2 lần:
+ Lần đầu → xóa thành công.
+ Lần sau → không tìm thấy dữ liệu.

1.4 HTTP Status Code (Mã trạng thái) (Lý thuyết)

Nhóm Ý nghĩa Ví dụ
1xx Thông tin 100 Continue
2xx Thành công 200 OK, 201 Created
3xx Chuyển hướng 301 Moved, 304 Not Modified
4xx Lỗi client 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found
5xx Lỗi server 500 Internal Server Error
Mã Tên Khi nào trả về Ví dụ
200 OK GET, PUT, DELETE thành công Lấy danh sách users
201 Created POST thành công Tạo user mới
204 No Content Xóa thành công (không trả dữ liệu) DELETE user
400 Bad Request Dữ liệu gửi lên không hợp lệ Thiếu trường name, email sai định dạng
401 Unauthorized Chưa đăng nhập Gọi API yêu cầu login
403 Forbidden Đã login nhưng không có quyền User thường cố xóa user khác
404 Not Found Không tìm thấy tài nguyên GET /users/999
405 Method Not Allowed Gửi sai phương thức POST vào endpoint chỉ hỗ trợ GET
500 Internal Server Error Lỗi từ phía server Code bị lỗi, database die

PHẦN 2: SERVLET CHO API

2.1 Servlet là gì? (Lý thuyết)

Định nghĩa Servlet là một class Java chạy trên server, nhận request từ client và trả về response.

Vòng đời của Servlet:

1. Container tải Servlet
         ↓
2. Gọi constructor → Tạo instance
         ↓
3. Gọi init() → 1 LẦN DUY NHẤT
         ↓
4. Gọi service() → MỖI REQUEST
    ├─ doGet()  → Xử lý GET
    ├─ doPost() → Xử lý POST
    ├─ doPut()  → Xử lý PUT
    └─ doDelete() → Xử lý DELETE
         ↓
5. Gọi destroy() → KHI DỪNG ỨNG DỤNG
Method Thời điểm gọi Mục đích
init() 1 lần khi Servlet được tải Khởi tạo tài nguyên (kết nối DB, đọc config)
service() Mỗi request Phân loại request (GET/POST/...)
doGet() Request GET Xử lý lấy dữ liệu
doPost() Request POST Xử lý thêm dữ liệu
doPut() Request PUT Xử lý cập nhật
doDelete() Request DELETE Xử lý xóa
destroy() 1 lần khi Servlet bị hủy Giải phóng tài nguyên

3.2 Cách đọc thông tin request trong Servlet

@WebServlet("/demo")
public class DemoServlet extends HttpServlet {
    
    protected void doPost(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        
        // ===== 1. ĐỌC REQUEST LINE =====
        String method = request.getMethod();           // GET, POST, PUT, DELETE
        String uri = request.getRequestURI();          // /demo
        String url = request.getRequestURL().toString(); // http://localhost:8080/demo
        String protocol = request.getProtocol();       // HTTP/1.1
        
        // ===== 2. ĐỌC QUERY STRING (dành cho GET) =====
        // Ví dụ: /demo?name=An&age=25
        String name = request.getParameter("name");    // "An"
        String age = request.getParameter("age");      // "25"
        
        // ===== 3. ĐỌC PATH VARIABLE (từ URL pattern) =====
        // Cần cấu hình @WebServlet("/users/*")
        String pathInfo = request.getPathInfo();       // /1 nếu URL là /users/1
        
        // ===== 4. ĐỌC HEADERS =====
        String contentType = request.getHeader("Content-Type");
        String userAgent = request.getHeader("User-Agent");
        String auth = request.getHeader("Authorization");
        
        // ===== 5. ĐỌC BODY (dành cho POST, PUT) =====
        StringBuilder sb = new StringBuilder();
        String line;
        try (BufferedReader reader = request.getReader()) {
            while ((line = reader.readLine()) != null) {
                sb.append(line);
            }
        }
        String body = sb.toString();  // Nội dung JSON gửi lên
        
        // ===== 6. ĐỌC COOKIE =====
        Cookie[] cookies = request.getCookies();
        
        // ===== 7. ĐỌC SESSION =====
        jakarta.servlet.http.HttpSession session = request.getSession();
        
        // Trả về response để kiểm tra
        response.setContentType("application/json");
        PrintWriter out = response.getWriter();
        out.print("{\"method\":\"" + method + "\", \"uri\":\"" + uri + "\"}");
    }
}

4.3 Lớp DBConnection

package com.example.api.util;

import java.io.InputStream;
import java.sql.*;
import java.util.Properties;

public class DBConnection {
    private static String url, username, password;

    static {
        try (InputStream input = DBConnection.class.getClassLoader()
                .getResourceAsStream("db.properties")) {
            Properties props = new Properties();
            props.load(input);
            url = props.getProperty("db.url");
            username = props.getProperty("db.username");
            password = props.getProperty("db.password");
            Class.forName(props.getProperty("db.driver"));
        } catch (Exception e) {
            throw new RuntimeException("Lỗi kết nối DB!");
        }
    }

    public static Connection getConnection() throws SQLException {
        return DriverManager.getConnection(url, username, password);
    }

    public static void close(Connection conn, PreparedStatement stmt, ResultSet rs) {
        try { if (rs != null) rs.close(); } catch (SQLException e) {}
        try { if (stmt != null) stmt.close(); } catch (SQLException e) {}
        try { if (conn != null) conn.close(); } catch (SQLException e) {}
    }
}

PHẦN 5: GET - Lấy dữ liệu

1.1 GET là gì? (Lý thuyết)
GET là phương thức HTTP dùng để lấy dữ liệu từ server.

Đặc điểm Mô tả Ví dụ
An toàn Không làm thay đổi dữ liệu trên server Lấy danh sách user không làm mất user
Idempotent Gọi 1 lần hay nhiều lần đều cho kết quả giống nhau GET /users/1 luôn trả về user có ID=1
Có thể cache Trình duyệt có thể lưu lại kết quả Trang danh sách sản phẩm có thể được cache
Dữ liệu gửi qua URL Tham số được gắn trên URL /users?name=An&age=25
Giới hạn độ dài URL có giới hạn (khoảng 2000 ký tự) Không nên gửi dữ liệu lớn qua GET

Khi nào dùng GET?
NÊN dùng GET khi:

  • Lấy danh sách dữ liệu (GET /users)
  • Lấy chi tiết một bản ghi (GET /users/5)
  • Tìm kiếm dữ liệu (GET /users?keyword=an), Lọc dữ liệu (GET/products?category=phone&minPrice=100)
KHÔNG dùng GET khi:
  • Thêm dữ liệu mới (dùng POST)
  • Cập nhật dữ liệu (dùng PUT/PATCH), Xóa dữ liệu (dùng DELETE)
  • Gửi mật khẩu, thông tin nhạy cảm
  • Gửi dữ liệu lớn (ảnh, file)

1.2 Các dạng GET API thường gặp

Dạng URL Mục đích Lấy tham số bằng
Lấy tất cả GET /users Lấy danh sách tất cả user Không có tham số
Lấy theo ID GET /users/5 Lấy user có ID=5 @PathVariable (path info)
Lấy theo tham số GET /users?name=An Tìm user có tên "An" @RequestParam (query string)
Lấy có phân trang GET /users?page=1&size=10 Lấy trang 1, mỗi trang 10 user @RequestParam

package com.example.api.servlet;

import java.io.IOException;
import java.io.PrintWriter;

import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloServlet extends HttpServlet {

    @Override
    protected void doGet(
            HttpServletRequest request,
            HttpServletResponse response)
            throws ServletException, IOException {

        // Bước 1: Thiết lập kiểu dữ liệu trả về là JSON
        response.setContentType("application/json");
        response.setCharacterEncoding("UTF-8");

        // Bước 2: Lấy đối tượng để ghi dữ liệu
        PrintWriter out = response.getWriter();

        // Bước 3: Tạo JSON
        String json =
                "{\"message\": \"Hello from API!\", \"status\": \"success\"}";

        // Bước 4: Ghi JSON vào response
        out.print(json);
        out.flush();
    }
}

Test: http://localhost:8080/YourProject/hello

Gson là thư viện của Google dùng để chuyển đổi giữa Java Object và JSON.

JSON → Object gson.fromJson(json, Class) Chuyển chuỗi JSON thành Java object

Tại sao dùng Gson thay vì tự tạo JSON bằng tay?

Tiêu chí Tự tạo JSON (StringBuilder) Dùng Gson Dòng code 10-20 dòng 1 dòng Dễ sai Rất dễ (quên dấu ,, ", escape) Không sai Xử lý null Phải tự kiểm tra Tự động Xử lý nested object Rất phức tạp Tự động Bảo trì Khó (thêm trường phải sửa code) Dễ (chỉ thêm vào class)

Ví dụ 2: API nhận tham số từ URL (cách cũ)

@WebServlet("/greet")
public class GreetServlet extends HttpServlet {
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("application/json");
        response.setCharacterEncoding("UTF-8");
        String name = request.getParameter("name");
        if (name == null || name.trim().isEmpty()) name = "Guest";
        PrintWriter out = response.getWriter();
        String json = "{\"message\": \"Xin chào " + name + "!\", \"status\": \"success\"}";
        out.print(json); out.flush();
    }
}

Với Gson

class GreetResponse {
    private String message; private String status;
    public GreetResponse(String message, String status) { this.message = message; this.status = status; }
    public String getMessage() { return message; }
    public String getStatus() { return status; }
}
@WebServlet("/greet")
public class GreetServlet extends HttpServlet {
    private final Gson gson = new Gson();
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        response.setContentType("application/json"); response.setCharacterEncoding("UTF-8");
        String name = request.getParameter("name");
        if (name == null || name.trim().isEmpty()) name = "Guest";
        GreetResponse greetResponse = new GreetResponse("Xin chào " + name + "!", "success");
        String json = gson.toJson(greetResponse);
        PrintWriter out = response.getWriter(); out.print(json); out.flush();
    }
}

PRODUCTDAO (CÁC PHƯƠNG THỨC TRUY VẤN)

package com.example.api.dao;

import com.example.api.model.Product;
import com.example.api.util.DBConnection;
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class ProductDAO {

    // 1. Lấy tất cả sản phẩm
    public List<Product> getAllProducts() {
        List<Product> products = new ArrayList<>();
        String sql = "SELECT * FROM products ORDER BY id ASC";
        try (Connection conn = DBConnection.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql);
             ResultSet rs = stmt.executeQuery()) {
            while (rs.next()) products.add(mapRow(rs));
        } catch (SQLException e) { e.printStackTrace(); }
        return products;
    }

    // 2. Lấy sản phẩm theo ID
    public Product getProductById(int id) {
        String sql = "SELECT * FROM products WHERE id = ?";
        try (Connection conn = DBConnection.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql)) {
            stmt.setInt(1, id);
            ResultSet rs = stmt.executeQuery();
            if (rs.next()) return mapRow(rs);
        } catch (SQLException e) { e.printStackTrace(); }
        return null;
    }

    // 3. Tìm kiếm theo tên (chứa keyword)
    public List<Product> searchByName(String keyword) {
        List<Product> products = new ArrayList<>();
        String sql = "SELECT * FROM products WHERE name LIKE ?";
        try (Connection conn = DBConnection.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql)) {
            stmt.setString(1, "%" + keyword + "%");
            ResultSet rs = stmt.executeQuery();
            while (rs.next()) products.add(mapRow(rs));
        } catch (SQLException e) { e.printStackTrace(); }
        return products;
    }

    // 4. Lấy phân trang (page bắt đầu từ 1, mỗi trang size bản ghi)
    public List<Product> getProductsByPage(int page, int size) {
        List<Product> products = new ArrayList<>();
        int offset = (page - 1) * size;
        String sql = "SELECT * FROM products ORDER BY id ASC LIMIT ? OFFSET ?";
        try (Connection conn = DBConnection.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql)) {
            stmt.setInt(1, size);
            stmt.setInt(2, offset);
            ResultSet rs = stmt.executeQuery();
            while (rs.next()) products.add(mapRow(rs));
        } catch (SQLException e) { e.printStackTrace(); }
        return products;
    }

    // 5. Đếm tổng số bản ghi (dùng cho phân trang)
    public int countAll() {
        String sql = "SELECT COUNT(*) FROM products";
        try (Connection conn = DBConnection.getConnection();
             PreparedStatement stmt = conn.prepareStatement(sql);
             ResultSet rs = stmt.executeQuery()) {
            if (rs.next()) return rs.getInt(1);
        } catch (SQLException e) { e.printStackTrace(); }
        return 0;
    }

    // Helper: chuyển ResultSet thành Product
    private Product mapRow(ResultSet rs) throws SQLException {
        Product p = new Product();
        p.setId(rs.getInt("id"));
        p.setName(rs.getString("name"));
        p.setPrice(rs.getDouble("price"));
        p.setQuantity(rs.getInt("quantity"));
        p.setCategory(rs.getString("category"));
        return p;
    }
}

PRODUCTSERVLET (XỬ LÝ 4 DẠNG GET)

package com.example.api.servlet;

import java.io.IOException;
import java.io.PrintWriter;
import java.util.List;
import com.google.gson.Gson;
import com.example.api.dao.ProductDAO;
import com.example.api.model.Product;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

@WebServlet("/products/*")
public class ProductServlet extends HttpServlet {

    private ProductDAO productDAO = new ProductDAO();
    private Gson gson = new Gson();

    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {

        response.setContentType("application/json");
        response.setCharacterEncoding("UTF-8");
        PrintWriter out = response.getWriter();

        String pathInfo = request.getPathInfo();  // Lấy phần sau /products

        // ========== 1. GET /products (lấy tất cả) ==========
        if (pathInfo == null || "/".equals(pathInfo)) {
            List<Product> products = productDAO.getAllProducts();
            response.setStatus(HttpServletResponse.SC_OK);
            out.print(gson.toJson(products));
            out.flush();
            return;
        }

        // ========== 2. GET /products?name=xxx (tìm theo tên) ==========
        String nameParam = request.getParameter("name");
        if (nameParam != null && !nameParam.trim().isEmpty()) {
            List<Product> products = productDAO.searchByName(nameParam);
            response.setStatus(HttpServletResponse.SC_OK);
            out.print(gson.toJson(products));
            out.flush();
            return;
        }

        // ========== 3. GET /products?page=1&size=10 (phân trang) ==========
        String pageParam = request.getParameter("page");
        String sizeParam = request.getParameter("size");
        if (pageParam != null && sizeParam != null) {
            try {
                int page = Integer.parseInt(pageParam);
                int size = Integer.parseInt(sizeParam);
                List<Product> products = productDAO.getProductsByPage(page, size);
                int total = productDAO.countAll();
                int totalPages = (int) Math.ceil((double) total / size);
                // Gói response metadata
                PageResponse wrapper = new PageResponse(products, page, size, total, totalPages);
                response.setStatus(HttpServletResponse.SC_OK);
                out.print(gson.toJson(wrapper));
                out.flush();
                return;
            } catch (NumberFormatException e) {
                response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
                out.print("{\"error\":\"page và size phải là số\"}");
                out.flush();
                return;
            }
        }

        // ========== 4. GET /products/{id} (lấy theo ID) ==========
        try {
            // pathInfo có dạng "/5" → bỏ dấu / đầu tiên
            int id = Integer.parseInt(pathInfo.substring(1));
            Product product = productDAO.getProductById(id);
            if (product != null) {
                response.setStatus(HttpServletResponse.SC_OK);
                out.print(gson.toJson(product));
            } else {
                response.setStatus(HttpServletResponse.SC_NOT_FOUND);
                out.print("{\"error\":\"Không tìm thấy sản phẩm có ID " + id + "\"}");
            }
            out.flush();
            return;
        } catch (NumberFormatException e) {
            response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
            out.print("{\"error\":\"ID không hợp lệ\"}");
            out.flush();
        }
    }

    // Inner class để gói response phân trang
    class PageResponse {
        private List<Product> data;
        private int page;
        private int size;
        private int total;
        private int totalPages;

        public PageResponse(List<Product> data, int page, int size, int total, int totalPages) {
            this.data = data;
            this.page = page;
            this.size = size;
            this.total = total;
            this.totalPages = totalPages;
        }
        // Getters (cần để Gson serialize)
        public List<Product> getData() { return data; }
        public int getPage() { return page; }
        public int getSize() { return size; }
        public int getTotal() { return total; }
        public int getTotalPages() { return totalPages; }
    }
}

PHẦN 6: POST - Thêm dữ liệu

PHẦN 1: LÝ THUYẾT VỀ POST

1.1 POST là gì? POST là phương thức HTTP dùng để gửi dữ liệu lên server nhằm tạo mới một tài nguyên.

Đặc điểm Mô tả Ví dụ
Không an toàn Làm thay đổi dữ liệu trên server Thêm user mới làm tăng số lượng user
Không idempotent Gửi 2 lần → tạo 2 bản ghi khác nhau POST /users 2 lần tạo 2 user khác nhau
Không cache Trình duyệt không lưu lại kết quả
Dữ liệu gửi trong Body Dữ liệu nằm trong body request, không hiện trên URL An toàn hơn cho mật khẩu
Không giới hạn độ dài Có thể gửi dữ liệu lớn (ảnh, file)

✅ NÊN dùng POST khi: Thêm mới một bản ghi, Đăng nhập, Gửi form dữ liệu, Upload file.
❌ KHÔNG dùng POST khi: Lấy dữ liệu (dùng GET), Cập nhật toàn bộ (dùng PUT), Xóa dữ liệu (dùng DELETE).

1.2 So sánh POST và GET

Tiêu chí GET POST
Mục đích Lấy dữ liệu Tạo dữ liệu mới
Dữ liệu gửi Qua URL (query string) Trong Body
Hiện trên URL Có (lộ thông tin) Không
Giới hạn dữ liệu Có (2000 ký tự) Không
Cache Có thể cache Không cache
Idempotent Có Không
An toàn Có (không thay đổi dữ liệu) Không

public boolean insertProduct(Product product) {
    String sql = "INSERT INTO products (name, price, quantity, category) VALUES (?, ?, ?, ?)";
    try (Connection conn = DBConnection.getConnection();
         PreparedStatement stmt = conn.prepareStatement(sql, Statement.RETURN_GENERATED_KEYS)) {
        
        stmt.setString(1, product.getName());
        stmt.setDouble(2, product.getPrice());
        stmt.setInt(3, product.getQuantity());
        stmt.setString(4, product.getCategory());
        
        int affectedRows = stmt.executeUpdate();
        
        if (affectedRows > 0) {
            ResultSet generatedKeys = stmt.getGeneratedKeys();
            if (generatedKeys.next()) {
                product.setId(generatedKeys.getInt(1));
            }
            return true;
        }
        return false;
    } catch (SQLException e) {
        e.printStackTrace();
        return false;
    }
}

public boolean updateProduct(Product product) {
    String sql = "UPDATE products SET name = ?, price = ?, quantity = ?, category = ? WHERE id = ?";
    try (Connection conn = DBConnection.getConnection();
         PreparedStatement stmt = conn.prepareStatement(sql)) {
        stmt.setString(1, product.getName());
        stmt.setDouble(2, product.getPrice());
        stmt.setInt(3, product.getQuantity());
        stmt.setString(4, product.getCategory());
        stmt.setInt(5, product.getId());
        int affectedRows = stmt.executeUpdate();
        return affectedRows > 0;
    } catch (SQLException e) {
        e.printStackTrace();
        return false;
    }
}

PHẦN 4: THÊM PHƯƠNG THỨC doPut VÀO PRODUCTSERVLET

@Override
protected void doPut(HttpServletRequest request, HttpServletResponse response)
        throws ServletException, IOException {

    response.setContentType("application/json");
    response.setCharacterEncoding("UTF-8");
    PrintWriter out = response.getWriter();

    String pathInfo = request.getPathInfo();

    if (pathInfo == null || "/".equals(pathInfo)) {
        response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
        out.print("{\"error\":\"Thiếu ID sản phẩm cần cập nhật\"}");
        out.flush();
        return;
    }

    int id;
    try {
        id = Integer.parseInt(pathInfo.substring(1));
    } catch (NumberFormatException e) {
        response.setStatus(HttpServletResponse.SC_BAD_REQUEST);
        out.print("{\"error\":\"ID không hợp lệ\"}");
        out.flush();
        return;
    }

    StringBuilder sb = new StringBuilder();
    String line;
    try (BufferedReader reader = request.getReader()) {
        while ((line = reader.readLine()) != null) {
            sb.append(line);
        }
    }
    String jsonBody = sb.toString();

    Product productData = gson.fromJson(jsonBody, Product.class);
    productData.setId(id);

    Product existing = productDAO.getProductById(id);
    if (existing == null) {
        response.setStatus(HttpServletResponse.SC_NOT_FOUND);
        out.print("{\"error\":\"Không tìm thấy sản phẩm có ID " + id + "\"}");
        out.flush();
        return;
    }

    boolean updated = productDAO.updateProduct(productData);

    if (updated) {
        response.setStatus(HttpServletResponse.SC_OK);
        out.print(gson.toJson(productData));
    } else {
        response.setStatus(HttpServletResponse.SC_INTERNAL_SERVER_ERROR);
        out.print("{\"error\":\"Lỗi cập nhật sản phẩm\"}");
    }
    out.flush();
}

PHẦN 5: GIẢI THÍCH CODE THEO TỪNG BƯỚC

Bước Mô tả Câu lệnh
1 Set Content-Type response.setContentType("application/json")
2 Lấy ID từ URL String pathInfo = request.getPathInfo()
3 Parse ID int id = Integer.parseInt(pathInfo.substring(1))
4 Đọc body JSON request.getReader()
5 Chuyển JSON → Object productData = gson.fromJson(jsonBody, Product.class)
6 Gán ID vào object productData.setId(id)
7 Kiểm tra tồn tại (tùy chọn) productDAO.getProductById(id)
8 Cập nhật database productDAO.updateProduct(productData)
9 Trả về response status 200 + JSON sản phẩm mới

PHẦN 6: CÁC STATUS CODE TRONG PUT
200 OK (cập nhật thành công), 400 Bad Request (thiếu ID hoặc ID không hợp lệ), 404 Not Found (không tìm thấy sản phẩm), 500 Internal Server Error (lỗi database).

PHẦN 7: KIỂM TRA API PUT VỚI POSTMAN
