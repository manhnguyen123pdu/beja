PHẦN 1: TỔNG QUAN VỀ WEB VÀ SERVLET

1.1 MÔ HÌNH CLIENT-SERVER LÀ GÌ?

Định nghĩa:

Mô hình Client-Server là kiến trúc mạng trong đó:

  • Client (máy khách) gửi yêu cầu
  • Server (máy chủ) nhận yêu cầu, xử lý và gửi phản hồi

Sơ đồ hoạt động:

┌─────────┐ Gửi yêu cầu ┌─────────┐ │ Client │ ──────────────────→ │ Server │ │(Browser)│ ←───────────────── │ (Web) │ └─────────┘ Nhận phản hồi └─────────┘

Ví dụ thực tế:

  • Bạn mở trình duyệt, gõ https://google.com
  • Trình duyệt (client) gửi yêu cầu đến máy chủ Google
  • Máy chủ Google xử lý và gửi trang web về
  • Trình duyệt hiển thị trang web cho bạn

1.2 CÁC THÀNH PHẦN TRONG WEB

Thành phần Mô tả Ví dụ
Client Trình duyệt, ứng dụng di động Chrome, Firefox, Postman
Server Máy chủ chứa ứng dụng web Tomcat, Jetty, Nginx
HTTP Giao thức giao tiếp Gửi request, nhận response
HTML/CSS/JS Nội dung hiển thị Giao diện website

1.3 CÁC LOẠI ỨNG DỤNG WEB

Loại Mô tả Ví dụ
Static Web Nội dung cố định, ai xem cũng giống nhau Trang giới thiệu công ty
Dynamic Web Nội dung thay đổi theo người dùng, dữ liệu Facebook, Shopee, Google
Ứng dụng Web động là mục tiêu của chúng ta trong khóa học này.

2: HTTP REQUEST & HTTP RESPONSE

2.1 HTTP LÀ GÌ?

Định nghĩa:

HTTP (HyperText Transfer Protocol) là giao thức truyền tải siêu văn bản, được dùng để giao tiếp giữa Client và Server.

Đặc điểm:

  • Hoạt động theo mô hình request-response
  • Stateless (không lưu trạng thái)
  • Cổng mặc định: 80 (HTTP), 443 (HTTPS)

2.2 HTTP REQUEST (YÊU CẦU)

Định nghĩa:

HTTP Request là thông tin mà Client gửi lên Server.

Cấu trúc một HTTP Request:

METHOD /path HTTP/version Header1: value1 Header2: value2 Body (nếu có)

Ví dụ thực tế:

POST /login HTTP/1.1 Host: example.com User-Agent: Mozilla/5.0 Content-Type: application/x-www-form-urlencoded Content-Length: 27 username=john&password=123

Các thành phần chính:

Thành phần Mô tả Ví dụ
Method Phương thức HTTP GET, POST, PUT, DELETE
Path Đường dẫn tài nguyên /login, /products
Version Phiên bản HTTP HTTP/1.1, HTTP/2
Headers Thông tin bổ sung Host, User-Agent
Body Dữ liệu gửi lên Form data, JSON

Các phương thức HTTP phổ biến:

Method Mô tả Ví dụ
GET Lấy dữ liệu Xem danh sách sản phẩm
POST Thêm dữ liệu mới Đăng ký tài khoản
PUT Cập nhật toàn bộ Sửa thông tin
PATCH Cập nhật một phần Sửa mật khẩu
DELETE Xóa dữ liệu Xóa sản phẩm

2.3 HTTP RESPONSE (PHẢN HỒI)

Định nghĩa:

HTTP Response là thông tin Server gửi lại Client sau khi xử lý request.

Cấu trúc một HTTP Response:

HTTP/version status_code status_text Header1: value1 Header2: value2 Body (nếu có)

Ví dụ thực tế:

HTTP/1.1 200 OK Content-Type: text/html Content-Length: 128 <html> <body> <h1>Chào mừng bạn!</h1> </body> </html>

Các thành phần chính:

Thành phần Mô tả Ví dụ
Status Code Mã trạng thái 200, 404, 500
Status Text Mô tả trạng thái OK, Not Found
Headers Thông tin bổ sung Content-Type
Body Nội dung trả về HTML, JSON

Các mã trạng thái HTTP thường gặp:

Mã Ý nghĩa Mô tả
200OKThành công
201CreatedTạo thành công
301Moved PermanentlyChuyển hướng vĩnh viễn
302FoundChuyển hướng tạm thời
400Bad RequestYêu cầu sai
401UnauthorizedChưa đăng nhập
403ForbiddenKhông có quyền
404Not FoundKhông tìm thấy
500Internal Server ErrorLỗi server
503Service UnavailableDịch vụ tạm dừng

3: GIỚI THIỆU SERVLET

3.1 SERVLET LÀ GÌ?

Định nghĩa:

Servlet là một chương trình Java chạy trên server, nhận request từ client và trả về response.

Nói dễ hiểu:

  • Servlet là class Java đặc biệt
  • Nó xử lý các request HTTP (GET, POST, ...)
  • Nó tạo ra nội dung động (HTML, JSON, ...)

Lịch sử:

  • Trước Servlet: viết CGI (Common Gateway Interface) bằng C/Perl → phức tạp
  • Servlet ra đời (1997) → dùng Java, dễ hơn
  • JSP (1999) → viết HTML dễ hơn

3.2 TẠI SAO CẦN SERVLET?

Vấn đề Giải pháp từ Servlet
Cần xử lý request từ client Servlet cung cấp API để đọc request
Cần tạo nội dung động Servlet có thể viết HTML trong code
Cần kết nối database Servlet có thể gọi JDBC
Cần quản lý session Servlet cung cấp API session

3.3 VÍ DỤ VỀ SERVLET

HelloServlet.java
import java.io.*; import javax.servlet.*; import javax.servlet.http.*; public class HelloServlet extends HttpServlet { @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { // Set kiểu nội dung trả về resp.setContentType("text/html"); // Lấy PrintWriter để ghi response PrintWriter out = resp.getWriter(); // Ghi HTML out.println("<html>"); out.println("<body>"); out.println("<h1>Hello, World!</h1>"); out.println("</body>"); out.println("</html>"); } }

Giải thích:

  • extends HttpServlet: class này là Servlet
  • doGet(): xử lý request GET
  • HttpServletRequest: chứa thông tin request
  • HttpServletResponse: để ghi response

4: SERVLET CONTAINER & TOMCAT

4.1 SERVLET CONTAINER LÀ GÌ?

Định nghĩa:

Servlet Container (còn gọi là Web Container) là môi trường chạy các Servlet.

Vai trò:

  • Quản lý vòng đời Servlet (khởi tạo, gọi, hủy)
  • Nhận request từ client và chuyển cho Servlet phù hợp
  • Quản lý đa luồng (mỗi request là một thread)
  • Cung cấp các dịch vụ: session, security, JSP

Sơ đồ hoạt động:

┌──────────────┐ │ Browser │ └──────┬───────┘ │ HTTP Request ┌──────▼───────┐ │ Tomcat │ │ (Container) │ │ ┌─────────┐ │ │ │Servlet1 │ │ │ ├─────────┤ │ │ │Servlet2 │ │ │ └─────────┘ │ └──────┬───────┘ │ HTTP Response ┌──────▼───────┐ │ Browser │ └──────────────┘

4.2 CÁC SERVLET CONTAINER PHỔ BIẾN

Tên Mô tả Phù hợp
Apache Tomcat Mã nguồn mở, phổ biến Học tập, doanh nghiệp nhỏ
Jetty Nhẹ, nhúng được Microservices
GlassFish Hỗ trợ đầy đủ Java EE Doanh nghiệp lớn
WildFly (JBoss) Mạnh mẽ Doanh nghiệp lớn

Trong khóa học này, chúng ta dùng Apache Tomcat.

4.3 APACHE TOMCAT

Định nghĩa:

Tomcat là một servlet container mã nguồn mở do Apache phát triển.

Đặc điểm:

  • Miễn phí, dễ cài đặt
  • Hỗ trợ Servlet, JSP, EL, WebSocket
  • Được dùng rộng rãi trong học tập và doanh nghiệp

Cấu trúc thư mục Tomcat:

tomcat/ ├── bin/ # Script khởi động ├── conf/ # Cấu hình (server.xml, context.xml) ├── lib/ # Các thư viện chung ├── logs/ # File log ├── temp/ # File tạm ├── webapps/ # Nơi deploy ứng dụng └── work/ # File biên dịch tạm

PHẦN 4: TẠO PROJECT SERVLET ĐẦU TIÊN

4.1 Tạo Dynamic Web Project (Không dùng Maven)

Bước 1: Tạo Project mới

  • Vào menu File > New > Dynamic Web Project
  • Nếu không thấy, chọn File > New > Other... > Tìm "Dynamic Web Project"

Bước 2: Cấu hình Project

  • Project name: ServletDemo
  • Target runtime: Chọn Apache Tomcat v9.0 (đã thêm ở trên)
  • Configuration: Chọn Default Configuration for Apache Tomcat v9.0
  • Context root: ServletDemo (mặc định)
  • Content directory: src/main/webapp (nên đặt theo chuẩn Maven)

Bước 3: Cấu hình thêm

  • Nhấn Next 2 lần
  • Tích chọn Generate web.xml deployment descriptor
  • Nhấn Finish

4.2 Tạo Project với Maven (Khuyến nghị)

Bước 1: Tạo Maven Project

  • Vào menu File > New > Maven Project
  • Cửa sổ New Maven Project hiện ra
  • Không tích "Create a simple project" (bỏ chọn nếu đang tích)
  • Nhấn Next >

Bước 2: Chọn Archetype

  • Trong ô Filter, gõ: webapp
  • Chọn maven-archetype-webapp (Group Id: org.apache.maven.archetypes)
  • Nhấn Next >

Bước 3: Nhập thông tin Project

  • Group Id: com.example (thường là tên công ty/domain ngược)
  • Artifact Id: servlet-demo (tên project)
  • Version: 1.0-SNAPSHOT (mặc định)
  • Package: com.example.controller (package chứa code)
  • Nhấn Finish

Cấu trúc project sau khi tạo:

Project Structure
servlet-demo/ ├── pom.xml # File cấu hình Maven ├── src/ │ ├── main/ │ │ ├── java/ # Chứa code Java │ │ │ └── (sẽ tạo package ở đây) │ │ └── webapp/ # Chứa tài nguyên web │ │ ├── WEB-INF/ │ │ │ └── web.xml # Deployment Descriptor │ │ └── index.jsp # Trang mặc định │ └── test/ # Chứa code test │ └── java/ └── target/ # Thư mục biên dịch (tự động tạo)

6.1 Tạo Package

Bước 1: Tạo thư mục nguồn

Bước 2: Tạo Package

  • Chuột phải vào src/main/java
  • Chọn New > Package
  • Nhập tên package: com.example.controller
  • Nhấn Finish

6.2 Tạo Servlet Class

Bước 1: Tạo Class mới

  • Chuột phải vào package com.example.controller
  • Chọn New > Class
  • Nhập Name: HelloServlet
  • Superclass: Nhấn Browse và tìm javax.servlet.http.HttpServlet
  • Nhấn Finish

Bước 2: Viết code cho Servlet

Hello.java
package demo1.controller; import java.io.IOException; import javax.servlet.ServletException; import javax.servlet.annotation.WebServlet; import javax.servlet.http.HttpServlet; import javax.servlet.http.HttpServletRequest; import javax.servlet.http.HttpServletResponse; @WebServlet("/hello") public class Hello extends HttpServlet { @Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException { resp.getWriter().append("demo"); } }

6.3 Giải thích code chi tiết

Thành phần Giải thích
@WebServlet("/hello") Annotation đánh dấu class là Servlet và mapping với URL /hello
extends HttpServlet Kế thừa từ HttpServlet để có các phương thức xử lý HTTP
doGet() Phương thức xử lý request GET
HttpServletRequest req Đối tượng chứa thông tin request từ client
HttpServletResponse resp Đối tượng để gửi response về client
resp.getWriter() Lấy đối tượng để ghi nội dung response

PHẦN 8: PHÂN TÍCH CHI TIẾT CÁC THÀNH PHẦN

1. Khai báo Package

Package declaration
package demo1.controller;

Chức năng: Xác định package (gói) chứa class này.

Ý nghĩa: Class Hello nằm trong thư mục demo1/controller. Điều này giúp quản lý mã nguồn theo mô hình phân lớp.

2. Các lệnh Import

Import statements
import java.io.IOException; import javax.servlet.ServletException; import javax.servlet.annotation.WebServlet; import javax.servlet.http.HttpServlet; import javax.servlet.http.HttpServletRequest; import javax.servlet.http.HttpServletResponse;

Chức năng: Import các thư viện cần thiết để Servlet có thể hoạt động.

Chi tiết:

  • java.io.IOException: Hỗ trợ xử lý lỗi vào/ra
  • javax.servlet.*: Các thư viện cốt lõi của Servlet
  • javax.servlet.http.*: Các thư viện xử lý HTTP

3. Annotation

Servlet annotation
@WebServlet("/hello")

Đây là điểm đặc biệt quan trọng trong Servlet 3.0+ (thay thế cho cách cũ là khai báo trong file web.xml).

Chức năng: Khai báo URL pattern để ánh xạ request từ trình duyệt vào Servlet này.

Ý nghĩa: Khi người dùng truy cập vào địa chỉ: http://localhost:8080/[context-path]/hello, thì Servlet này sẽ được kích hoạt.

4. Khai báo Class

Class declaration
public class Hello extends HttpServlet
  • public class Hello: Tên class trùng với tên file (Hello.java)
  • extends HttpServlet: Kế thừa từ class cha HttpServlet

5. Phương thức doGet

doGet method
@Override protected void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException
  • @Override: Ghi đè phương thức của class cha
  • protected void doGet: Xử lý request GET
  • HttpServletRequest req: Đối tượng request
  • HttpServletResponse resp: Đối tượng response

6. Xử lý logic và Phản hồi

Response handling
resp.getWriter().append("demo");

Phân tích:

  • resp.getWriter(): Lấy đối tượng PrintWriter để ghi dữ liệu
  • .append("demo"): Ghi chuỗi "demo" vào response

Tổng kết Luồng hoạt động:

  1. Client gửi request tới URL: /hello
  2. Server thấy annotation @WebServlet("/hello") → gọi class Hello
  3. Server tạo đối tượng HttpServletRequest và HttpServletResponse
  4. Server gọi phương thức doGet (vì request là GET)
  5. Code trong doGet chạy: ghi chữ "demo" vào response
  6. Server gửi response về client
