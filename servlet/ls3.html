1.1 Tổng quan về JSP

Định nghĩa JSP (JavaServer Pages) là công nghệ cho phép nhúng mã Java vào trong trang HTML để tạo ra các trang web động. JSP được biên dịch thành Servlet trước khi thực thi.

Tại sao cần JSP?

Hãy xem vấn đề khi chỉ dùng Servlet:

Ví dụ Servlet thuần - Rất khó đọc và bảo trì 
protected void doGet(HttpServletRequest request, HttpServletResponse response) 
        throws ServletException, IOException {
    response.setContentType("text/html");
    PrintWriter out = response.getWriter();
    
    out.println("<html>");
    out.println("<head><title>Danh sách người dùng</title></head>");
    out.println("<body>");
    out.println("<h1>Danh sách người dùng</h1>");
    out.println("<table border='1'>");
    out.println("<tr><th>ID</th><th>Họ tên</th><th>Email</th></tr>");
    
    for (User user : userList) {
        out.println("<tr>");
        out.println("<td>" + user.getId() + "</td>");
        out.println("<td>" + user.getName() + "</td>");
        out.println("<td>" + user.getEmail() + "</td>");
        out.println("</tr>");
    }
    
    out.println("</table>");
    out.println("</body>");
    out.println("</html>");
}

  • Bảo trì rất khó khăn

1.4 Servlet Hello_JSP

Định nghĩa Servlet là class Java xử lý request từ client. Trong mô hình MVC, Servlet đóng vai trò Controller - nhận request, xử lý logic, lấy dữ liệu, rồi chuyển tiếp (forward) sang JSP để hiển thị.

Cú pháp Servlet cơ bản

Cú pháp Servlet 
@WebServlet("/duong-dan")
public class TenServlet extends HttpServlet {
    
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        // Xử lý GET request
    }
    
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        // Xử lý POST request
    }
}
Giải thích cú pháp:
- @WebServlet("/duong-dan"): Ánh xạ URL đến Servlet này
- extends HttpServlet: Kế thừa lớp HttpServlet
- doGet(): Xử lý HTTP GET method
- doPost(): Xử lý HTTP POST method
- request: Chứa thông tin request từ client
- response: Dùng để gửi response về client

Ví dụ nhỏ: HelloServlet

File: src/main/java/com/example/servlet/HelloServlet.java 
package com.example.servlet;

import java.io.IOException;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;

@WebServlet("/hello")
public class HelloServlet extends HttpServlet {
    
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) 
            throws ServletException, IOException {
        
        // 1. Tạo dữ liệu (giả sử lấy từ database)
        String message = "Xin chào từ Servlet!";
        String currentTime = java.time.LocalDateTime.now().toString();
        
        // 2. Đặt dữ liệu vào request attribute để gửi sang JSP
        request.setAttribute("servletMessage", message);
        request.setAttribute("servletTime", currentTime);
        
        // 3. Chuyển tiếp (forward) sang JSP
        request.getRequestDispatcher("/WEB-INF/views/hello.jsp")
               .forward(request, response);
    }
}
Giải thích ví dụ:
- @WebServlet("/hello"): Khi người dùng truy cập http://localhost:8080/my_jsp/hello thì Servlet này được gọi
- request.setAttribute("servletMessage", message): Lưu dữ liệu vào request với tên "servletMessage" để JSP lấy ra
- request.getRequestDispatcher("/WEB-INF/views/hello.jsp"): Lấy đối tượng Dispatcher để chuyển tiếp đến JSP
- .forward(request, response): Thực hiện chuyển tiếp (forward) - dữ liệu trong request được giữ nguyên

Giải thích JSP nhận dữ liệu:
- <%@ page ... %>: Directive - cấu hình cho trang JSP
- <% ... %>: Scriptlet - code Java thực thi
- request.getAttribute("servletMessage"): Lấy dữ liệu từ request do Servlet gửi sang
- <%= message %>: Expression - in giá trị ra HTML

                    ↓

                    ↓
8. Trả về browser

- Loading: Class loader tải file .class vào bộ nhớ
- Instantiation: Tạo instance của Servlet (gọi constructor)
- Initialization: Gọi jspInit() - thường dùng để kết nối database, đọc file cấu hình
- Request Handling: Gọi _jspService() - xử lý request, sinh HTML
- Destroy: Gọi jspDestroy() - đóng kết nối, giải phóng tài nguyên

Định nghĩa JSP cho phép kết hợp HTML và Java để tạo trang web động. Java code được nhúng vào trong HTML thông qua các tag đặc biệt.

2.2 Directive - Chỉ dẫn JSP

Định nghĩa Directive cung cấp thông tin chỉ dẫn cho JSP container về cách xử lý trang JSP. Directive không tạo ra output trực tiếp.

Cú pháp

<%@ directive_name attribute1="value1" attribute2="value2" %>

Các loại Directive

Ví dụ nhỏ: page directive

<%@ page language="java" contentType="text/html; charset=UTF-8"

Giải thích các thuộc tính page directive:
- language: Ngôn ngữ script (mặc định: java)
- contentType: Kiểu nội dung trả về (mặc định: text/html)
- pageEncoding: Mã hóa trang JSP (mặc định: ISO-8859-1)
- import: Import package/class
- errorPage: Trang xử lý lỗi
- isErrorPage: Có phải trang lỗi không? (mặc định: false)
- session: Có tạo session không? (mặc định: true)
- buffer: Kích thước buffer (mặc định: 8kb)
- autoFlush: Tự động flush buffer (mặc định: true)
- isThreadSafe: Có an toàn luồng không? (mặc định: true)
- info: Thông tin về JSP
- isELIgnored: Có bỏ qua EL không? (mặc định: false)

2.3 Declaration - Khai báo

Định nghĩa Declaration dùng để khai báo các biến instance và phương thức cho class Servlet được sinh ra từ JSP. Code trong declaration trở thành thành viên (member) của class.

Cú pháp

<%!
    // Khai báo biến instance
    private int counter = 0;
    private String appName = "My App";
    
    // Khai báo phương thức
    public String formatDate(java.util.Date date) {
        return new java.text.SimpleDateFormat("dd/MM/yyyy").format(date);
    }
    
    public int add(int a, int b) {
        return a + b;
    }
%>
Giải thích
- Đặc điểm: Chuyển thành code bên ngoài method _jspService()
- Phạm vi: Toàn bộ class, dùng chung cho mọi request
- Thời điểm khởi tạo: Khi JSP được tải (1 lần)
- Dùng chung: Giữa các request, giữa các user

Ví dụ nhỏ: declaration cơ bản

Định nghĩa Scriptlet là đoạn mã Java được thực thi mỗi khi có request. Code trong scriptlet được đặt bên trong method _jspService().

Cú pháp

        // Code lặp

        // Code có thể gây lỗi

- Đặc điểm: Chuyển thành code bên trong method _jspService()

- Có thể dùng: Các implicit objects (request, response, out, session...)
- Có thể dùng: Biến và phương thức từ declaration

package com.example.servlet;

import com.example.model.User;
import jakarta.servlet.ServletException;
import jakarta.servlet.annotation.WebServlet;
import jakarta.servlet.http.HttpServlet;
import jakarta.servlet.http.HttpServletRequest;
import jakarta.servlet.http.HttpServletResponse;
import jakarta.servlet.http.HttpSession;

import java.io.IOException;
import java.util.ArrayList;
import java.util.List;

@WebServlet("/users/*")
public class UserServlet extends HttpServlet {
    
    // Dữ liệu giả lập (thực tế nên dùng database)
    private static List<User> users = new ArrayList<>();
    
    static {
        users.add(new User("admin", "Administrator", "admin@system.com", 30));
        users.add(new User("john", "John Doe", "john@example.com", 25));
        users.add(new User("jane", "Jane Smith", "jane@example.com", 28));
    }
    
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        
        String pathInfo = request.getPathInfo();
        
        if ("/list".equals(pathInfo)) {
            // Hiển thị danh sách user
            request.setAttribute("users", users);
            request.getRequestDispatcher("/WEB-INF/views/user_list.jsp")
                   .forward(request, response);
                   
        } else if (pathInfo != null && pathInfo.startsWith("/edit/")) {
            // Sửa user
            Long id = Long.parseLong(pathInfo.substring(6));
            User user = users.stream()
                    .filter(u -> u.getId() != null && u.getId().equals(id))
                    .findFirst()
                    .orElse(null);
            request.setAttribute("user", user);
            request.getRequestDispatcher("/WEB-INF/views/user_form.jsp")
                   .forward(request, response);
                   
        } else if ("/add".equals(pathInfo)) {
            // Thêm mới user
            request.setAttribute("user", new User());
            request.getRequestDispatcher("/WEB-INF/views/user_form.jsp")
                   .forward(request, response);
        }
    }
    
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response)
            throws ServletException, IOException {
        
        String action = request.getParameter("action");
        
        if ("create".equals(action)) {
            // Thêm user mới
            User user = new User();
            user.setUsername(request.getParameter("username"));
            user.setPassword(request.getParameter("password"));
            user.setFullName(request.getParameter("fullName"));
            user.setEmail(request.getParameter("email"));
            user.setAge(Integer.parseInt(request.getParameter("age")));
            user.setRole(request.getParameter("role"));
            user.setId((long) (users.size() + 1));
            
            users.add(user);
            
            request.getSession().setAttribute("message", "Thêm user thành công!");
            response.sendRedirect(request.getContextPath() + "/users/list");
            
        } else if ("update".equals(action)) {
            // Cập nhật user
            Long id = Long.parseLong(request.getParameter("id"));
            User existingUser = users.stream()
                    .filter(u -> u.getId().equals(id))
                    .findFirst()
                    .orElse(null);
            
            if (existingUser != null) {
                existingUser.setFullName(request.getParameter("fullName"));
                existingUser.setEmail(request.getParameter("email"));
                existingUser.setAge(Integer.parseInt(request.getParameter("age")));
                existingUser.setRole(request.getParameter("role"));
                
                if (request.getParameter("password") != null && 
                    !request.getParameter("password").trim().isEmpty()) {
                    existingUser.setPassword(request.getParameter("password"));
                }
            }
            
            request.getSession().setAttribute("message", "Cập nhật user thành công!");
            response.sendRedirect(request.getContextPath() + "/users/list");
            
        } else if ("delete".equals(action)) {
            // Xóa user
            Long id = Long.parseLong(request.getParameter("id"));
            users.removeIf(u -> u.getId().equals(id));
            
            request.getSession().setAttribute("message", "Xóa user thành công!");
            response.sendRedirect(request.getContextPath() + "/users/list");
        }
    }
}
